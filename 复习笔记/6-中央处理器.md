## 6.1 🏛️ CPU概述

### 6.1.1 CPU的功能

CPU就是一台**"取指令-执行指令"的机器**，它干五件事：

| 功能 | 英文 | 通俗理解 |
|:----|:----|:---------|
| **数据加工** 🔢 | Data Processing | 算算术、做逻辑运算 |
| **程序控制** 🎮 | Program Control | 决定下一条执行哪条指令 |
| **操作控制** 🎛️ | Operation Control | 把指令翻译成控制信号指挥各部件 |
| **时序控制** ⏱️ | Timing Control | 确保所有操作按正确的顺序发生 |
| **异常控制** 🚨 | Exception Control | 处理突发情况（如除零、缺页） |

### 6.1.2 CPU的内部结构

```
                    CPU
    ┌──────────────────────────────────────────────┐
    │                控制器 (CU)                    │
    │  ┌──────┐  ┌──────┐  ┌────────────────────┐  │
    │  │  PC  │  │  IR  │  │  微操作控制信号     │  │
    │  │程序计│  │指令寄│  │  发生电路           │  │
    │  │数器  │  │存器  │  │  (硬布线或微程序)   │  │
    │  └──────┘  └──────┘  └────────────────────┘  │
    ├──────────────────────────────────────────────┤
    │               数据通路 (DP)                   │
    │  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────────┐ │
    │  │通用寄│  │  ALU │  │  MAR │  │  MDR     │ │
    │  │存器组│  │运算器│  │地址寄│  │数据寄存器│ │
    │  │(32个)│  │      │  │存器  │  │          │ │
    │  └──────┘  └──────┘  └──────┘  └──────────┘ │
    │              ←内部总线→                     │
    └──────────────────────────────────────────────┘
```

### 6.1.3 主要寄存器

| 寄存器 | 英文名称 | 作用 | 类比 |
|:------|:--------|:-----|:-----|
| **PC** 📍 | Program Counter | **存放下一条指令的地址**（自动+1） | 书签→读到的页 |
| **IR** 📋 | Instruction Register | **存放当前正在执行的指令** | 正在做的菜谱 |
| **MAR** 📮 | Memory Address Register | **存放要访问的内存地址** | 快递地址 |
| **MDR** 📦 | Memory Data Register | **存放从内存读出/写入的数据** | 快递包裹 |
| **通用寄存器** 🗂️ | General Purpose Registers | 临时存数据（MIPS有32个$0~$31） | 便签纸 |
| **PSW** 🏳️ | Program Status Word | **存放运算标志**（零、负、进位等） | 成绩单 |

### 6.1.4 两种控制器实现方式

| 方式 | 怎么实现 | 速度 | 特点 | 典型应用 |
|:----|:---------|:----:|:------|:--------|
| **硬布线控制器** 🔌 | 用**逻辑门电路**搭出来 | 🚀**快** | 设计复杂、固定、难改 | RISC (MIPS) |
| **微程序控制器** 📜 | 用**ROM存微指令** | 🐢慢 | 设计容易、灵活易改 | CISC (x86) |

## 6.2 🛤️ 数据通路 ⭐⭐⭐

### 6.2.1 什么是数据通路？

> **数据通路 = 数据在CPU内部流动的"道路系统" + 控制信号 = "交警"🚦**

```
数据通路由以下组成：
① 存储元件：寄存器（PC、IR、寄存器组等）
② 运算元件：ALU（加法器、移位器等）
③ 连接线路：总线（数据流动的"道路"）
④ 控制信号：决定数据往哪流（"红绿灯"）
```

### 6.2.2 单总线数据通路

> **所有部件都连到一条总线上，同一时间只能传输一个数据**

**以 MIPS `ADD $rd, $rs, $rt`为例（寄存器加法）：**

```
时钟1: Rs → 总线 → A锁存器    （送第一个操作数）
时钟2: Rt → 总线 → B锁存器    （送第二个操作数）
时钟3: A锁存器+B锁存器 → ALU → 总线 → Rd（计算+写回）

需要3个时钟周期！
```

**为什么需要两个锁存器？**
```
因为总线同一时间只能传一个数据
如果直接把Rs和Rt同时给ALU → 总线冲突！
所以先锁存一个，再传另一个
```

### 6.2.3 双总线数据通路

> **两条总线，可以同时传两个数据**

```
时钟1: Rs → 总线A + Rt → 总线B → ALU → 结果暂存
时钟2: 暂存结果 → 写入Rd

只需要2个时钟周期！
锁存器也只需要1个（暂存结果用）
```

### 6.2.4 三总线数据通路

> **两条输入总线 + 一条输出总线，完全并行**

```
时钟1: Rs → 总线A, Rt → 总线B, ALU计算, 结果 → 总线C → Rd

只需要1个时钟周期！
无需锁存器！
```

### 6.2.5 数据通路对比

| 结构 | 锁存器数量 | 一次加法耗时 | 并行能力 |
|:----:|:---------:|:-----------:|:--------:|
| **单总线** 🔌 | 2个 | **3个时钟周期** | 差 |
| **双总线** 🔌🔌 | 1个 | **2个时钟周期** | 中 |
| **三总线** 🔌🔌🔌 | 0个 | **1个时钟周期** | 好 |

> **结论：总线越多，速度越快，硬件越复杂**

### 6.2.6 MIPS指令在单总线CPU上的执行流程 ⭐⭐

#### 取指令（所有指令都一样）

```
T1: PC → 总线 → MAR        （把PC中的地址送到地址寄存器）
T2: 读内存 → MDR            （从内存中读出指令）
T3: MDR → 总线 → IR         （指令送入指令寄存器）
T4: PC + 4 → 总线 → PC      （PC指向下一条指令）
```

> **取指需要4个时钟周期**

#### lw $rt, offset($rs) — 从内存读数据到寄存器

```
额外需要3个时钟周期：
T5: Rs → 总线 → A锁存器
T6: A锁存器 + 立即数(符号扩展) → ALU → 总线 → MAR
T7: 读内存 → MDR → 总线 → Rt
```

#### sw $rt, offset($rs) — 从寄存器写数据到内存

```
额外需要2个时钟周期：
T5: Rs → 总线 → A锁存器
T6: A锁存器 + 立即数 → ALU → 总线 → MAR
T7: Rt → 总线 → MDR → 写内存
```

#### beq $rs, $rt, offset — 条件跳转

```
T5: Rs → 总线 → A锁存器
T6: Rt → 总线 → B锁存器
T7: A-B → ALU → 判断结果
    如果相等：PC + 偏移量 → 总线 → PC
    如果不相等：不操作
```

#### add $rd, $rs, $rt — 寄存器加法

```
T5: Rs → 总线 → A锁存器
T6: Rt → 总线 → B锁存器
T7: A + B → ALU → 总线 → Rd
```

## 6.3 ⏰ 指令周期 ⭐⭐

### 6.3.1 三级时序系统

> CPU的时间单位是分级的，就像时间分"年/月/日"

```
时钟周期 = 最小时钟单位（一次脉冲）                ← "秒"
机器周期 = 完成一个基本操作的时间（通常=访存时间）  ← "分钟"
指令周期 = 执行一条完整指令的时间                  ← "小时"
```

**关系：**
```
指令周期 = 若干个机器周期
每个机器周期 = 若干个时钟周期
```

### 6.3.2 一个指令周期包含

```
取指周期 (Fetch)     ← 从内存取指令
  └→ 间址周期 (Indirect)  ← 如果指令需要，取操作数地址（可选）
    └→ 执行周期 (Execute)  ← 真正执行指令
      └→ 中断周期 (Interrupt) ← 如果收到中断请求（可选）
```

**不同指令的指令周期不同：**
```
空操作(NOP)：取指 → 执行(空)                     ← 最短
ADD 寄存器：取指 → 执行(寄存器加法)                ← 短
LW 从内存：取指 → 执行(计算地址→读内存→写入寄存器)  ← 长
MUL 乘法：  取指 → 执行(计算→等待→写回)            ← 最长
```

### 6.3.3 定长 vs 变长指令周期

| 方式 | 机器周期数 | 节拍数 | 优点 | 缺点 |
|:----|:---------:|:------:|:-----|:-----|
| **定长** 📏 | 固定（所有指令相同） | 固定（按最复杂的来） | 控制简单 | **浪费**（简单指令也被拖慢） |
| **变长** 📐 | 按指令实际需要 | 可按需增减 | 高效 | 控制复杂 |

> 现代CPU基本都用变长指令周期

## 6.4 🔌 硬布线控制器 ⭐⭐

### 6.4.1 基本思想

> 用**逻辑门电路**（与门、或门、非门）直接搭出电路的控制器
> 输入 = 指令 + 时序信号 + 状态
> 输出 = 控制信号

### 6.4.2 控制信号的本质

```
每条指令对应一套"微操作序列"
每个微操作 = 一个控制信号

例：lw指令（加载）
取指阶段：
  PCout → MARin     (PC地址→MAR)
  MemRead           (读内存)
  MDRout → IRin     (指令→IR)
  PC+4 → PC         (PC加4)

执行阶段：
  Rsout → Ain       (Rs→A锁存器)
  ALUadd            (ALU做加法：Rs+偏移量)
  Zout → MARin      (结果→MAR)
  MemRead           (读内存)
  MDRout → Rt_in    (数据→Rt)
```

### 6.4.3 控制信号的逻辑表达式

> **每个控制信号 = f(当前指令, 当前节拍)**

举例：
```
MAR = MIF · T2                     （取指阶段：PC→MAR）
    + (lw+sw+addi) · MEX · T1      （执行阶段：ALU结果→MAR）
    + ...

MDR = MIF · T3                     （取指：MDR→IR传给CPU）
    + lw · MEX · T3                （lw加载：MDR→寄存器）
    + ...

解释：
  MIF = 取指周期
  MEX = 执行周期
  T1, T2, T3 = 节拍
  lw, sw, addi = 指令译码输出
```

### 6.4.4 设计步骤

```
① 分析数据通路 → 画出指令周期流程图
② 确定每条指令在各阶段的微操作
③ 写出每个控制信号的逻辑表达式
④ 用门电路实现这些表达式
```

### 6.4.5 特点

| 优点 ✅ | 缺点 ❌ |
|:--------|:---------|
| 速度**最快**（纯硬件） | 设计复杂繁琐 |
| 适合RISC（简单指令集） | 修改困难（要改电路） |
| | 指令多了没法做 |

## 6.5 📜 微程序控制器 ⭐⭐

### 6.5.1 核心理念

> **把控制信号"写"到一个ROM（控制存储器）里，当做程序来运行！**

```
就像：
硬布线控制器 = 用电路直接实现"1+1=2"（搭死电路）
微程序控制器 = 查"加法表"（查表，可以改）

对比：
普通程序（软件）：在内存中运行 → 处理数据
微程序（固件）：在控制存储器中运行 → 产生控制信号
```

### 6.5.2 结构

```
指令 → 微地址 → 控制存储器(ROM) → 微指令 → 微操作控制信号
                   ↓
                存了所有微程序
```

**控制存储器的内容：**
```
地址 0: 取指微程序 （所有指令共用！）
地址 1: ADD的微程序
地址 2: SUB的微程序
地址 3: LW的微程序
...
```

### 6.5.3 微指令格式

```
┌──────────────────────────────────────────────────┐
│              微指令 (控存中一行)                    │
├──────────────────────┬───────────────────────────┤
│  操作控制字段（微命令）│    顺序控制字段（下址/判别）  │
├──────────────────────┼───────────────────────────┤
│ 每个bit控制一个开关   │  指出下一条微指令地址      │
│ =1 打开这个开关       │  或根据条件判断下地址      │
│ =0 关闭这个开关       │                          │
└──────────────────────┴───────────────────────────┘
```

### 6.5.4 微程序执行过程

```
取指周期：
  ① 根据PC中的地址 → 从主存取指令 → 存入IR
  ② 根据IR中的操作码 → 映射到对应的微程序入口地址
  ③ 从控存中取第一条微指令 → 产生控制信号
  ④ 根据下址字段取下一条微指令
  ⑤ 重复直到这个指令的所有微指令执行完毕
  ⑥ 回到取指周期的微程序入口
```

### 6.5.5 水平型 vs 垂直型微指令

| 对比 | 水平型 📏 | 垂直型 📐 |
|:----|:---------|:---------:|
| **并行能力** | 🚀**强**（可以同时控制多个开关） | 🐢弱（一次只能控制一个） |
| **微指令字长** | **长**（几百位） | 短（几十位） |
| **微程序长度** | 短（并行所以步骤少） | 长（串行所以步骤多） |
| **执行速度** | 🚀**快** | 🐢慢 |
| **编码方式** | 直接控制/字段直接编码 | 类似机器指令编码 |
| **使用现状** | ✅**主流** | ❌已淘汰 |

> **通俗理解：**
> - 水平型 = 一个人同时干10件事（效率高，但大脑复杂）
> - 垂直型 = 一次干1件事，干10次（效率低，但简单）

### 6.5.6 硬布线 vs 微程序 终极对比 ⭐

| 对比项 | 硬布线控制器 🔌 | 微程序控制器 📜 |
|:-------|:--------------|:---------------|
| **本质** | 组合逻辑+时序逻辑的硬件电路 | 存储逻辑（ROM存微指令） |
| **速度** 🚀 | **快**（下一个时钟直接出结果） | 慢（要读ROM） |
| **设计** | 复杂繁琐 | **简单规整** |
| **修改** | 难（要重新设计电路） | **易**（改ROM内容即可） |
| **成本** | 高（复杂指令集成本爆炸） | 低 |
| **灵活性** | 差 | **好**（可升级） |
| **典型应用** | **RISC** (MIPS, ARM) | **CISC** (x86) |

## 6.6 🏗️ 单周期 vs 多周期 MIPS CPU

### 单周期CPU

> **每条指令都在一个时钟周期内完成**

```
优点：
  CPI = 1（每个指令一个周期）→ 直观

缺点：
  时钟周期必须按"最慢的指令"来设计
  比如乘法要10ns → 所有指令都只能按10ns来
  → 一条ADD本来1ns就能完成，也要等10ns → 浪费！
  
  而且：很多硬件在一个周期内只用一次（如ALU只在执行阶段用）
  → 硬件利用率很低
```

### 多周期CPU

> **每条指令分成多个步骤，每个步骤一个时钟周期**

```
优点：
  时钟周期可以很短（按最"快"的步骤来）
  不同指令可以用不同周期数
  硬件可以在不同时间复用（ALU先算地址再算下一条）

缺点：
  CPI > 1（CPI可变）
  控制逻辑更复杂

例子：
  lw需要5个周期
  add需要4个周期
  平均CPI约4
  但时钟频率可以比单周期高5倍 → 性能反而更好！
```

| 对比 | 单周期 | 多周期 |
|:----|:------|:-------|
| CPI | **=1** | >1（可变） |
| 时钟周期 | **长**（由最慢指令决定） | **短** |
| 硬件利用率 | 低（很多部件闲置） | **高**（部件复用） |
| 实际性能 | 一般 | **更好** |
| 可流水化？ | ❌ 不行 | ✅ **可以进一步流水** |

## 6.7 🚨 异常和中断处理

### 异常类型

| 类型 | 怎么发生 | 例子 |
|:----|:---------|:------|
| **故障 (Fault)** ⚠️ | 指令执行时出问题 | 缺页、除零、非法指令 |
| **自陷 (Trap)** 🪤 | **程序主动触发** | syscall（系统调用） |
| **终止 (Abort)** 💀 | **硬件严重故障** | 内存校验错、总线错误 |
| **外部中断** 📞 | **外设请求CPU注意** | 键盘按下了、网卡收到数据 |

### 异常处理流程

```
正常程序执行：
取指令 → 执行指令 → 取指令 → 执行指令 → ...

发生异常时：
取指令 → 执行指令 → !!故障/自陷/中断!!
                      │
                      ▼
                  关中断（不再接收新的中断）
                      │
                      ▼
                  保存断点（把当前PC保存到EPC）
                      │
                      ▼
                  跳转到异常处理程序入口
                      │
                      ▼
                  保护现场（保存通用寄存器到栈）
                      │
                      ▼
                  异常处理服务（根据原因处理）
                      │
                      ▼
                  恢复现场（从栈恢复寄存器）
                      │
                      ▼
                  开中断
                      │
                      ▼
                  异常返回（ERET指令，从EPC恢复PC）
```

### MIPS异常处理相关寄存器

```
EPC (Exception PC)：保存被异常打断的指令地址
Cause寄存器：记录异常原因（除零？缺页？系统调用？）
Status寄存器：中断使能位
BadVaddr寄存器：出错的地址（用于缺页异常）
```

```
CPU就是取执器，不断循环三件事：
取指→译码→执行，周而复始不停歇

数据通路是道路，总线越多路越宽
单总线用三时钟，三总线就一时钟
锁存器是临时站，数据到了先暂存

硬布线快设计难，逻辑门搭控制路
微程序简速度慢，控存取指令来翻译
水平并行效率高，垂直淘汰已过时

单周期CPI为一，但周期长效率低
多周期虽步骤多，时钟快了反更好
进一步能流水化，现代CPU都在用

异常中断要分清，内部异常外部断
保存断点EPC存，处理完后再返回
```

## ⭐ MIPS常用数据通路操作

| 操作 | 微操作序列（单总线） |
|:-----|:-------------------|
| **取指令** | PC→MAR → 读M[MAR]→MDR → MDR→IR → PC+4→PC |

| **sw** | Rs→A → A+imm→MAR → Rt→MDR → 写M[MAR] |

| **addi** | Rs→A → A+imm→ALUout → ALUout→Rt |
