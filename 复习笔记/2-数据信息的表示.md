## 2.1 💡 为什么计算机用二进制？

### 十进制 vs 二进制

| 进制 | 数字符号 | 优点 | 缺点 |
|:----:|:---------|:-----|:-----|
| **十进制** 🔟 | 0,1,2,...,9 | 人类习惯了 | 10种状态→电路复杂 |
| **二进制** 2️⃣ | **0和1** | 物理易实现、运算规则简单 | 写起来很长 |

**二进制物理实现方案：**
```
0 = 低电压(0V)    1 = 高电压(5V/3.3V)
0 = 开关断开      1 = 开关闭合
0 = 电容没电荷    1 = 电容有电荷
0 = 磁不朝这方向  1 = 磁朝这方向
```

> **💡 核心：** 只有两种状态的东西最容易做、最可靠！

### 二进制的运算规则

**加法**（最简单的！）：

二进制加法只有三条规则：
```
0 + 0 = 0
0 + 1 = 1
1 + 1 = 0（进位1）   ← 懂这三条，就会算所有二进制加法了！
```

**对比十进制加法规则：** 45种规则（0+0到9+9还要考虑进位）
→ 所以说二进制让电路设计**大大简化**了！

## 2.2 🔄 进制转换

### 2.2.1 常用进制

| 进制 | 符号 | 一位表示范围 | 缩写 |
|:----:|:----:|:-----------:|:----:|
| **二进制** 2️⃣ | 0,1 | 0~1 | B |
| **八进制** 8️⃣ | 0~7 | 0~7 | O |
| **十进制** 🔟 | 0~9 | 0~9 | D |
| **十六进制** 🔢 | 0~9,A~F | 0~15 | H |

### 2.2.2 进制转换方法 ⭐

#### ① 二进制 ↔ 八进制/十六进制

```
二进制 → 八进制：三位一组（从右往左）
  例：10 011 100₂ = 234₈
      ↓   ↓   ↓
      2   3   4

二进制 → 十六进制：四位一组（从右往左）
  例：1001 1100₂ = 9C₁₆
       ↓    ↓
       9    C
```

#### ② 十进制 → 二进制

> **整数部分：除2取余，从下往上**
> **小数部分：乘2取整，从上往下**

**整数转换示例：将 29 转为二进制**
```
29 ÷ 2 = 14 ... 余 1  ↑
14 ÷ 2 =  7 ... 余 0  │
 7 ÷ 2 =  3 ... 余 1  │
 3 ÷ 2 =  1 ... 余 1  │
 1 ÷ 2 =  0 ... 余 1  │
                         29₁₀ = 11101₂
```

**小数转换示例：将 0.625 转为二进制**
```
0.625 × 2 = 1.25 → 整数1  ↓
0.25  × 2 = 0.5  → 整数0  │
0.5   × 2 = 1.0  → 整数1  │
                          0.625₁₀ = 0.101₂
```

### 2.2.3 常用的2的幂（必须背下来！）📌

```
2⁰  = 1       2¹⁰ = 1024 ≈ 1K
2¹  = 2       2²⁰ = 1,048,576 ≈ 1M
2²  = 4       2³⁰ ≈ 1G
2³  = 8       2⁴⁰ ≈ 1T
2⁴  = 16      2⁵⁰ ≈ 1P
2⁵  = 32
2⁶  = 64
2⁷  = 128     ← 最重要的！ASCII码范围
2⁸  = 256     ← 一个字节的取值范围
2⁹  = 512
2¹⁰ = 1024    ← 1K
```

> **速算技巧：** 130 = 128+2 → 二进制 = 10000010₂
> 先把接近的2的幂找出来，再拆！

## 2.3 📊 机器码（重中之重！）⭐⭐⭐

### 为什么要研究机器码？

```
计算机不认识"+"和"-"
所以必须把数字的"正负号"也用0和1编码！

用最高位表示符号：0 = 正数, 1 = 负数
剩余位表示数值
```

### 2.3.1 四种机器码对比

假设机器字长**8位**（1个字节），看数字 **+5** 和 **-5** 的表示：

| 码制 | +5的表示 | -5的表示 | 零的表示 | 特点 |
|:----|:--------:|:--------:|:--------:|:------|
| **原码** 🏳️ | 0000 0101 | 1000 0101 | 0000 0000 / 1000 0000 | 直观，但有两个零 |
| **反码** 🙃 | 0000 0101 | 1111 1010 | 0000 0000 / 1111 1111 | 已淘汰 |
| **补码** ⭐ | 0000 0101 | 1111 1011 | **0000 0000** | **唯一零！最常用** |
| **移码** 📏 | 1000 0101 | 0111 1011 | 1000 0000 | 用于浮点数阶码 |

### 2.3.2 原码 (Sign-Magnitude)

**定义：**
```
正数：符号位=0，后面是数值的二进制
负数：符号位=1，后面是数值的二进制

例（8位）：
[+5]原 = 0000 0101
[-5]原 = 1000 0101
```

**缺点：**
```
① 有两个零：+0 = 0000 0000, -0 = 1000 0000
② 做加减法非常复杂！需要比较符号，决定用加法还是减法
  例：5 - 3 = 5 + (-3)
  原码直接加：0000 0101 + 1000 0011 = 1000 1000 = -8 ❌ 完全错误
  所以原码不能直接用加法器！
```

### 2.3.3 反码 (One's Complement)

**定义：**
```
正数：同原码
负数：原码的数值位按位取反

例（8位）：
[+5]反 = 0000 0101
[-5]反 = 1111 1010（0000 0101按位取反）
```

**缺点：** 仍然有两个零！

### 2.3.4 补码 (Two's Complement) ⭐⭐⭐

> **这是最最最重要的！计算机中的整数都用补码存储！**

**定义：**
```
正数：同原码
负数：反码 + 1

另一种更快的求法（扫描法）：
从右向左找第一个1，该位左边全部取反

例：求 -5 的补码
原码：1000 0101
反码：1111 1010
补码：1111 1011（反码+1）

扫描法：1000 0101 → 从右找第一个1（第0位）→ 左边取反 → 1111 1011
```

**为什么补码这么重要？**
```
因为补码可以把减法变成加法！

X - Y = X + (-Y的补码)

例：5 - 3 = 5 + (-3的补码)
  5 的补码：0000 0101
  -3的补码：1111 1101
  相加：  0000 0101 + 1111 1101 = 1 0000 0010
          ↑
        丢弃进位 → 0000 0010 = 2 ✅ 正确答案！
```

**补码的优点：**
| 优点 | 说明 |
|:----|:------|
| ✅ **唯一零** | 0000 0000 就是0，没有+0和-0 |
| ✅ **减法变加法** | 不用单独做减法器，一个加法器搞定 |
| ✅ **符号位参与运算** | 不用单独处理符号位 |

**补码的对称性缺点：**
```
8位补码范围：-128 ~ +127
正数比负数少一个！
因为 1000 0000 表示 -128，但+128需要9位才能表示

记住：补码中，负数的个数比正数多1
```

**符号位扩展（必考！）⭐⭐**

> **把8位补码变成16位补码：符号位复制到高位**

```
例：8位补码 1001 0011 = -109
   → 16位补码 1111 1111 1001 0011（高位全部填1）

例：8位补码 0100 0001 = +65
   → 16位补码 0000 0000 0100 0001（高位全部填0）

规则：正数补0，负数补1
```

### 2.3.5 移码 (Excess/Biased)

**定义：**
```
[X]移 = 2ⁿ + X     (n为数值位位数)

用途：主要用于浮点数的**阶码(指数)**表示
```

**特点：**
```
- 移码可以比较大小！这在浮点数中很重要
- 移码和补码的关系：符号位取反即得补码

例：n=7, [+5]移 = 1000 0101
            [-5]移 = 0111 1011
```

### 2.3.6 四种码的表示范围

**定点整数范围：**（n+1位，含1位符号位）

| 码制 | 范围 | 8位示例 |
|:----|:-----|:-------:|
| 原码 | `-(2ⁿ-1) ~ (2ⁿ-1)` | -127 ~ +127 |
| 反码 | `-(2ⁿ-1) ~ (2ⁿ-1)` | -127 ~ +127 |
| **补码** ⭐ | **`-2ⁿ ~ (2ⁿ-1)`** | **-128 ~ +127** |
| 移码 | `-2ⁿ ~ (2ⁿ-1)` | -128 ~ +127 |

**定点小数范围：**

| 码制 | 范围 | 8位示例 |
|:----|:-----|:-------:|
| 原码 | `-(1-2⁻ⁿ) ~ (1-2⁻ⁿ)` | -127/128 ~ 127/128 |
| 反码 | `-(1-2⁻ⁿ) ~ (1-2⁻ⁿ)` | -127/128 ~ 127/128 |
| **补码** ⭐ | **`-1 ~ (1-2⁻ⁿ)`** | **-1 ~ 127/128** |

## 2.5 🚀 浮点数 (IEEE 754标准) ⭐⭐⭐

### 为什么需要浮点数？

**现实中的数字跨度太大了：**
```
电子的质量：   0.0000000000000000000000000009g  ← 太小
太阳的质量：   2000000000000000000000000000000g  ← 太大
用定点数表示 → 不是溢出就是精度不够！
```

**解决方案：科学记数法**

```
就像物理学中的：
电子的质量 ≈ 9 × 10⁻²⁸ g
太阳的质量 ≈ 2 × 10³³ g

一个"符号" × "尾数" × "基数指数"
```

### IEEE 754 标准格式

```
N = (-1)^S × M × 2^E

S = 符号位（0正1负）
M = 尾数（有效数字的小数部分）
E = 阶码（指数，用移码表示）
```

**两种常用格式：**

```
单精度 float (32位)：
| S(1位) | E(8位) | M(23位) |
|   31   | 30~23  | 22~0    |

双精度 double (64位)：
| S(1位) | E(11位) | M(52位) |
|   63   | 62~52   | 51~0    |
```

### 浮点数的小秘密 🤫

> **规格化数要求尾数M在[1, 2)之间**
> 因为二进制规格化：小数点前一定是1 → 这个1被"隐含"了，不存！
> → 这样白嫖了一位精度！🎉

```
例：3.75₁₀ → 二进制 11.11 → 1.111 × 2¹
存的是：S=0, E=1+127=128, M=111000...
隐含的"1."不存，只存"111"
```

### 特殊值

| 情况 | S | E | M | 代表 |
|:----|:-:|:-:|:-:|:----:|
| **+0** | 0 | 0 | 0 | 真正的零 |
| **-0** | 1 | 0 | 0 | 也是零 |
| **+∞** | 0 | 全1(255) | 0 | 正无穷大 |
| **-∞** | 1 | 全1(255) | 0 | 负无穷大 |
| **NaN** | 任意 | 全1(255) | ≠0 | 不是数字(如0/0) |
| **非规格化数** | 0/1 | 0 | ≠0 | 接近零的小数 |

### 浮点数转换示例

**例：把 3.3 转为 IEEE 754单精度**

```
① 3.3 的二进制 = 11.010011001100110011...（无限循环小数！）
② 规格化：1.1010011001100110011... × 2¹
③ S = 0（正数）
④ E = 1 + 127 = 128 = 10000000₂
⑤ M = 10100110011001100110011（隐含的1去掉，取23位）

结果：0 10000000 10100110011001100110011
```

**例：反过来，把 IEEE 754单精度 变成十进制**

```
给定：0 10000010 11000000000000000000000

① S = 0 → 正数
② E = 10000010₂ = 130 → 真值 = 130 - 127 = 3
③ M = 1.11（加上隐含的1）
④ 值 = 1.11 × 2³ = 1110₂ = 14₁₀
```

### ⚠️ 浮点数的坑

| 坑 | 示例 | 原因 |
|:---|:-----|:------|
| **不精确** 🎭 | 3.3/1.1 ≠ 3.0 | 二进制无限循环，只能截断 |
| **不满足结合律** 🔀 | (a+b)+c ≠ a+(b+c) | 精度损失导致 |
| **上溢/下溢** 💥 | 太大或太小的数 | 超出可表示范围 |

## 2.7 ✅ 数据校验码 ⭐⭐⭐

### 为什么需要校验？

> 数据在传输过程中可能会被干扰：0变成1，1变成0
> **校验码 = 在数据中增加冗余信息，用来检测（甚至纠正）错误**

### 2.7.1 码距

**码距** = 两个合法编码之间不同的位数

```
例：00 和 11 的码距 = 2（双方全部不同）

一个检纠错能力的判定：
码距 = d，那么：
- 可以检测 d-1 个错误
- 可以纠正 (d-1)/2 个错误（向下取整）
```

| 能力 | 最小码距要求 |
|:----|:-----------:|
| 检测 e 个错误 | ≥ e + 1 |
| 纠正 t 个错误 | ≥ 2t + 1 |
| 同时纠正 t 个 + 检测 e 个 | ≥ e + t + 1 |

### 2.7.2 奇偶校验

**最简单的方法：**

```
在数据末尾加1位校验位
使整个数据（包括校验位）中1的个数是奇数或偶数

奇校验：使1的个数为奇数
偶校验：使1的个数为偶数

例：数据 1011010（其中有4个1）
偶校验位 = 0（已经有偶数个1了，加0保持偶数）
奇校验位 = 1（加1让1变成奇数个）
```

**优点：** 简单、高效 ✅
**缺点：** 只能检测奇数个错误，不能定位错误 ❌

### 2.7.3 海明码 ⭐⭐

> **比奇偶校验更高级：能发现错误，还能知道是"第几位"错了！**

**原理：** 用多个校验位，每个校验位"管"一组数据位，各组交叉重叠

**怎么确定校验位个数？**
```
海明不等式：2^r ≥ k + r + 1
r = 校验位数，k = 数据位数
```

**(7,4)海明码（4位数据+3位校验位）：**

```
数据位：D4 D3 D2 D1
校验位：P3 P2 P1

校验关系：
G1 = P1 ⊕ D1 ⊕ D2 ⊕ D4    ← P1校验 D1,D2,D4
G2 = P2 ⊕ D1 ⊕ D3 ⊕ D4    ← P2校验 D1,D3,D4
G3 = P3 ⊕ D2 ⊕ D3 ⊕ D4    ← P3校验 D2,D3,D4

如果 G3G2G1 = 000 → 没有错误
如果 G3G2G1 ≠ 000 → 这个值就是出错的位号！
```

> **通俗理解：** 海明码像做"交叉检查"
> - 检查员1：第1、2、4位是不是对的？
> - 检查员2：第1、3、4位是不是对的？
> - 检查员3：第2、3、4位是不是对的？
> - 告诉你是哪一组出问题 → 精确锁定错误位置！ 🎯

### 2.7.4 CRC循环冗余校验 ⭐⭐

> **一种更强大的校验方法，用除法（模2除法）来实现**

**核心思想：**
```
发送方：数据后面加一些校验位，使得整个编码能被一个"生成多项式"整除
接收方：用同样的多项式去除收到的数据
       - 余数为0 → 数据大概率正确
       - 余数不为0 → 数据出错了！
```

**具体步骤（以CRC-3为例）：**

```
数据：1010
生成多项式 G(x) = x³ + x + 1 → 二进制 1011

① 数据后面补3个0（次数=多项式最高次）→ 1010000
② 用1011做模2除法（不进位加法/不借位减法，即异或）
③ 得到余数 R(x)
④ 最终发送：1010 + R(x)（即 1010xxx）
```

**常用生成多项式：**

| 协议 | 多项式 | 二进制 |
|:----|:------|:------|
| **CRC-8** (1-Wire) | x⁸ + x⁵ + x⁴ + 1 | 100110001 |
| **CRC-16** (USB,蓝牙) | x¹⁶ + x¹⁵ + x² + 1 | 11000000000000101 |
| **CRC-32** (以太网) | x³² + ... | 100000100110000010001110110110111 |

> **💡 关键理解：** CRC的除法是**模2除法**，其实就是异或(XOR)运算
> 不注意这点容易算错！

### 校验方法对比

| 方法 | 码距 | 能检错？ | 能纠错？ | 额外开销 | 应用 |
|:----|:---:|:--------:|:--------:|:--------:|:-----|
| **奇偶校验** | 2 | ✅奇数错 | ❌ | 1位 | 串口通信 |
| **海明码** ⭐ | 3 | ✅ | ✅1位错 | log₂(数据位) | 内存ECC |
| **CRC** ⭐ | 4+ | ✅很强 | ❌ | 校验位长度 | 网络/存储 |
