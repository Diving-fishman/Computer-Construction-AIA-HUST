## 11.1 📋 DSP基本概念

### 11.1.1 什么是DSP？

```
DSP 双重含义：
  Digital Signal Processing  → 数字信号处理（技术/方法）
  Digital Signal Processor   → 数字信号处理器（芯片）⭐ 本章重点
```

### 11.1.2 DSP芯片的发展历程

```
1980年代    1990年代       2000年代        2010年代+
  │           │              │               │
早期DSP    高性能DSP    多核DSP/VLIW     KeyStone架构
TI TMS320   C6000系列    C647X系列       C665X/C667X
                                         异构计算
```

### 11.1.3 DSP芯片分类

| 分类方式 | 类型 | 特点 | 代表 |
|:--------|:-----|:-----|:------|
| **按数据格式** | **定点DSP** 🎯 | 整数运算，成本低 | TMS320C64x |
| | **浮点DSP** 🎯 | 硬件浮点运算，精度高 | TMS320C67x |
| **按用途** | **通用DSP** 🔧 | 可编程，灵活 | TI、ADI系列 |
| | **专用DSP** 📟 | 针对特定算法固化 | FFT专用芯片 |

> **注意：** 浮点DSP有浮点运算硬件，也可进行定点运算
> 定点DSP没有浮点运算硬件，浮点运算需要开发环境支持

### 11.1.4 DSP性能指标 ⭐

| 指标 | 含义 | 示例 |
|:----|:-----|:------|
| **指令周期** ⏱️ | 执行一条指令的时间 | TMS320LC549-80: 80MHz → 12.5ns |
| **MIPS** 🏃 | 百万条指令/秒 | C6416 @1GHz → **8000 MIPS** |
| **MOPS** 🏃 | 百万次操作/秒 | C6201 @200MHz → 2400 MOPS |
| **MFLOPS** 🔢 | 百万次浮点操作/秒 | ADSP-TS201S → **110.3 GFLOPS** |
| **Mbps** 📊 | 百万位/秒（总线吞吐率） | C6000 @200MHz → 800 MB/s |
| **MAC时间** ✖️➕ | 完成一次乘-累加的时间 | 大部分DSP可**单周期**完成 |
| **FFT时间** 📐 | 完成N点FFT的时间 | 取决于芯片性能 |

> **MAC（乘-累加）是DSP最核心的操作：** `y += a[i] × b[i]`
> 大多数DSP都能在**一个指令周期**内完成一次MAC！

## 11.2 🔬 DSP芯片的特点 ⭐⭐⭐

### 11.2.1 采用超级哈佛结构

> **传统冯·诺依曼：** 指令和数据共用一条总线 → 取指令和取数据不能同时进行
> **哈佛结构：** 程序存储器和数据存储器**分开**，各自独立总线 → 可同时取指令和取数据

```
冯·诺依曼结构：             哈佛结构：
┌──────────────┐          ┌─────────┐ 程序总线  ┌─────────┐
│ 统一存储器    │←──总线──→│ 程序存储器 │←────────→│         │
│ (程序+数据)  │          └─────────┘           │   CPU   │
└──────────────┘          ┌─────────┐ 数据总线  │         │
                          │ 数据存储器 │←────────→│         │
                          └─────────┘          └─────────┘

超级哈佛结构 → 程序和数据总线进一步分离，支持更多并行访问
```

### 11.2.2 采用多总线结构 ⭐

> **多总线 = 可同时进行取指令和多个数据存取操作**

```
TMS320C54x 内部有 P、C、D、E 等4组总线，每组都有地址总线和数据总线。
一个机器周期内可以完成：
  ✅ 从程序存储器中取一条指令
  ✅ 从数据存储器中读两个操作数
  ✅ 向数据存储器写一个操作数

→ 数据吞吐率极大提升！
```

### 11.2.3 采用流水线技术 ⭐

> 多条指令并行执行，不提高时钟频率就能减少执行时间

```
四级流水线示例：

          时钟1    时钟2    时钟3    时钟4    时钟5    时钟6
指令N    取指     译码     读数     执行
指令N+1           取指     译码     读数     执行
指令N+2                    取指     译码     读数     执行
指令N+3                             取指     译码     读数     执行

→ 每个时钟周期都有一条指令完成执行！
→ 流水线填满后，吞吐率 = 1条指令/周期
```

### 11.2.4 集成功能强大的CPU

```
DSP CPU内核通常包含：
  ✅ 算术逻辑运算单元(ALU)
  ✅ 累加器(ACC)
  ✅ 硬件乘法器 ⭐（核心！）
  ✅ 乘加单元(MAC)
  ✅ 桶型移位器
  ✅ 程序地址产生器
  ✅ 数据地址产生器

TMS320C64x：双数据通道 + 8个功能单元 → 单周期可同时执行8条指令！
```

### 11.2.5 集成专用的硬件乘法器 ⭐

> **这是DSP区别于通用CPU的最重要特征！**

| 处理器 | 乘法指令周期数 | 原因 |
|:-------|:------------:|:------|
| MCS-51单片机 | 4个周期 | 无硬件乘法器，用软件实现 |
| **通用DSP** 🚀 | **1个周期** | **有专用硬件乘法器** |

```
DSP的MAC（乘-加）操作：
  y += a[i] × b[i]   ← 滤波器/FFT的核心操作

在一个周期内完成：
  ① 乘法：a[i] × b[i]
  ② 加法：累加到y
  ③ 地址更新：i++

TMS320C6000系列有2个硬件乘法器 → 单周期可做2次乘法！
```

### 11.2.6 特殊的专用指令

```
复合指令：一条指令中完成多个操作

例：TMS320C24x的 MPYA（乘且累加前次乘积）
  执行MPYA后同时发生：
  (1) PC+1 → PC
  (2) ACC + PREG(前次乘积) → ACC  ← 累加前次结果
  (3) TREG × 数据存储器 → PREG   ← 同时做新乘法
  (4) AR1+1 → AR1                    ← 地址自增
  ...

→ 一条指令完成：累加 + 乘法 + 地址更新！
```

**其他专用特性：**
```
✅ 指令重复循环硬件：自动重复执行单条或一段指令
✅ 位翻转指令：用于FFT的位序重排
✅ 循环寻址指令：用于卷积运算
✅ 互锁指令：多片DSP间通信
```

### 11.2.7 嵌入强大硬件功能

```
DSP = 数字信号处理能力 + 嵌入式功能

片内外设：
  ┌────────────────────────────────┐
  │         DSP CPU 内核            │
  ├────────┬────────┬──────────────┤
  │ JTAG   │ 时钟    │ Bootloader    │
  │ 仿真接口 │ 管理    │               │
  ├────────┼────────┼──────────────┤
  │ 程序    │ 数据    │ 电源控制       │
  │ 存储器  │ 存储器  │               │
  ├────────┼────────┼──────────────┤
  │ 通信端口 │ 主机接口 │ 定时/计数器    │
  │        │ (HPI)  │               │
  ├────────┼────────┼──────────────┤
  │ 模拟I/O │ 编解码器 │               │
  └────────┴────────┴──────────────┘

→ 不用外扩很多器件就能组成独立应用系统！
```

### 11.3.2 图像处理DSP选型原则

| 考虑因素 | 选择建议 |
|:---------|:---------|
| **定点 vs 浮点** 🎯 | 图像处理一般选**定点**；动态范围/精度高选浮点（如密集FFT） |
| **内存大小** 💾 | **片上内存**访问速度快，偏向大内存 |
| **外设接口** 🔌 | 满足系统接口需求 |
| **功耗** ⚡ | 符合系统功耗预算 |
| **成本** 💰 | 满足性能前提下控制成本 |

```
┌──────────────────────────────────────────────┐
│                 C64x DSP                      │

│  │ CPU内核   │  │ 片上存储器 │  │  EDMA控制器 │  │
│  │ (8功能单元)│  │ L1+L2    │  │  (64通道)   │  │
│  └──────────┘  └──────────┘  └────────────┘  │
│  ┌──────────────────────────────────────────┐ │
│  │         片上集成外设                      │ │
│  │  EMIFA  │  EMIFB  │  McBSP  │  GPIO  ...│ │
│  └──────────────────────────────────────────┘ │
│  ┌──────────┐  ┌──────────┐                   │
│  │ 时钟管理   │  │ 启动配置   │                   │
│  └──────────┘  └──────────┘                   │
└──────────────────────────────────────────────┘
```

### 11.4.2 C64x内核结构

```
C64x CPU内核组成：
  ✅ 程序预取单元
  ✅ 指令分包单元
  ✅ 指令译码单元
  ✅ 双数据通路（A、B两组，各含4个功能单元）
  ✅ 64个32位寄存器（A组32个 + B组32个）
  ✅ 控制寄存器
  ✅ 控制逻辑
  ✅ JTAG仿真测试单元
  ✅ 中断控制逻辑
```

### 11.4.3 内核功能单元

**8个功能单元分为A/B两组：**

| 功能单元 | 功能 | 说明 |
|:---------|:-----|:------|
| **.L** 🧮 | 算术和逻辑运算(ALU) | 加减、与或非、比较 |
| **.S** 🔄 | 算术和逻辑运算(ALU) | 移位、位操作、分支 |
| **.M** ✖️ | **乘法运算** ⭐ | 单周期乘法，MAC核心 |
| **.D** 📍 | 数据寻址操作 | 寄存器与内存之间数据交换 |

> **VLIW（超长指令字）：**
> 每个指令周期，CPU将多个操作打包成一个"长指令"
> 同时分配给8个功能单元执行 → 单周期最多执行**8条指令**！

### 11.4.4 中断系统

| 特性 | 说明 |
|:-----|:------|
| **中断数量** | 支持16种不同优先级中断（INT_00~INT_15） |
| **最高优先级** | INT_00 → **RESET中断** |
| **最低优先级** | INT_15 |
| **不可屏蔽** | INT_00~INT_03（4个），包括NMI |
| **用户外部中断** | INT4~INT7 |
| **可屏蔽中断条件** | 最高优先级 + IER对应位=1 + GIE=0 + NMIE=1 |

**中断服务表IST：**
```
512Byte内存块，16个ISFP（中断服务取指包）组成
每个ISFP = 32Byte，可存放8条指令
超过8条 → 设置跳转指令跳转到其它部分
不足8条 → NOP填充
IST可搬移到任意256Byte边界对齐的存储单元
复位中断INT_00必须存放在地址0处
```

**中断响应过程：**
```
① 接收中断请求 → IFR对应位置1
② 中断确认（可屏蔽中断需满足条件）
③ 执行中断服务程序：
   保护现场（PC压栈）
   载入中断向量表
   执行ISR
   恢复现场（栈顶弹回PC）
   继续执行主程序
```

### 11.4.5 两级存储结构 ⭐

```
C64x 两级存储架构：

  CPU内核
     ↕       高速访问（Cache，用户不可直接访问）
  L1P(16KB)  L1D(16KB)     ← 靠近内核，速度最快
     ↕
  L2(1MB)                   ← 靠近外设，速度略慢，用户可访问
     ↕
  EDMA/外设                  ← DMA直接访问L2

  L2模式可配置，按字节单位统一编址
  DMA直接访问L2，内核直接访问L1
```

### 11.4.6 EDMA控制器 ⭐

> **EDMA = Enhanced Direct Memory Access（增强型直接存储器访问）**

| 特性 | 说明 |
|:-----|:------|
| **功能** | 外设与L2存储器之间的直接数据传输 |
| **独立性** | **独立于CPU内核工作** |
| **数据宽度** | 支持8bit、16bit、32bit |
| **通道数** | **64个独立通道** |
| **优先级** | 4个优先级级别 |

**EDMA的三种启动方式：**

| 方式 | 触发条件 | 适用场景 |
|:----|:---------|:---------|
| **CPU启动** 🖥️ | CPU写ESR寄存器 | 程序控制的主动数据传输 |
| **同步事件触发** ⚡ | 外设中断等事件 | **外设突发数据传输** |
| **链触发** 🔗 | 前一个EDMA完成 | 连续传输场景 |

**EDMA与CPU关系：**
```
✅ 双方可独立访问L2
✅ EDMA优先级低于CPU优先级
✅ EDMA的启动需要CPU设置
✅ EDMA传输完成后需向CPU递交中断信号
```

### 11.4.7 外部存储器接口EMIF ⭐

> **EMIF = External Memory Interface — DSP数据I/O的重要门户**

| 特性 | EMIFA | EMIFB |
|:-----|:-----:|:-----:|
| **位宽** | 64位 | 16位 |
| **功能** | 连接大容量高速存储器 | 连接低速外设 |

**支持的存储器类型：**
```
✅ 同步静态存储器(SBSRAM)
✅ 同步动态存储器(SDRAM)
✅ 异步存储器(FLASH/EEPROM等)
```

**EMIF控制寄存器：**

| 寄存器 | 功能 |
|:-------|:------|
| **GBLCTL** 全局控制 | 所有CE空间的公共参数设置 |
| **CExCTL** CE空间控制 | 对应CE空间的控制寄存器（x=0,1,2,3） |
| **CExSEC** CE空间二级控制 | 控制可编程同步存储器访问时序 |
| **SDCTL** SDRAM控制 | SDRAM全体CE空间参数 |
| **SDTIM** SDRAM时序控制 | SDRAM刷新周期 |
| **SDEXT** SDRAM扩展 | SDRAM扩展参数 |

## 11.5 💾 存储器接口技术

### 11.5.1 存储器分类

```
                          存储器
                    ┌──────┴──────┐
                    │              │
                易失性          非易失性
              (RAM)           (ROM/FLASH)
              ┌──┴──┐            │
           SRAM    DRAM       FLASH/PROM/EPROM/EEPROM
         (静态)    (动态)         ┌────┴────┐
                   需刷新        NOR       NAND
```

**图像处理系统中的存储器：**

| 类型 | 用途 | 要求 | 典型器件 |
|:----|:-----|:-----|:---------|
| **程序存储器** 📝 | 存储程序、FPGA配置、参数模板 | 非易失、稳定可靠、容量不大 | FLASH/EEPROM |
| **数据存储器** 📊 | 图像数据（帧缓存）、中间结果 | **速度快、容量大** | SRAM/DRAM |

### 11.5.2 程序存储器 — FLASH

**FLASH发展史：**
```
1984年 东芝舛冈富士雄博士发明FLASH
1988年 Intel推出首款NOR FLASH
1989年 东芝推出NAND FLASH
```

**NOR vs NAND FLASH：**

| 对比 | NOR FLASH | NAND FLASH |
|:----|:----------|:------------|
| **存储单元连接** | 并行连接 | 顺序连接 |
| **访问方式** | 随机存储（可字节寻址） | **按页访问** |
| **容量** | 相对小 | **容量大** |
| **读取速度** | **快**（支持XIP，代码可直接运行） | 慢（需要先读到RAM） |
| **写入速度** | 慢（字节编程） | 快（页编程） |
| **典型应用** | BIOS/UEFI、程序存储 | U盘、SSD、大容量存储 |

**SST39VF080 NOR FLASH：**
```
容量：1M × 8bit
扇区：4KB扇区，16个扇区组成64KB块
擦除：支持扇区/块/整片擦除
编程：只能逐个字节编程
特点：写操作需按特定协议，读操作有速度要求
```

### 11.5.3 数据存储器 — FIFO ⭐

> **FIFO = First In First Out（先进先出队列）**

```
FIFO基本模型：
            WCLK     RCLK
              │       │
     D[n-1:0] ─┤ FIFO ├─→ Q[n-1:0]
              │       │
            WEN#    REN#
            FF#     EF#
```

**FIFO的两种模式：**

| 模式 | 读时序 | 特点 |
|:----|:-------|:------|
| **标准模式** 📋 | 读使能有效→**下一周期**数据有效 | 延迟一个时钟 |
| **FWFT模式** 🚀 | 读使能有效→**当前周期**数据有效 | 零延迟（First Word First Through） |

**FIFO的使用场合：**

```
① 速度匹配与数据互连
   ADC ──→ FIFO ──→ DSP
   时钟域1        时钟域2
   → 解决两个设备速度差异，保持同步

② FPGA内部实现
   厂家提供FIFO的IP核
   图像处理频繁使用

③ ⚠️ FIFO使用要点：
   ✅ 判空/满标志，避免异常
   ❌ FIFO满时写入 → 满异常
   ❌ FIFO空时读出 → 空异常
   ❌ FIFO异常后只能通过复位恢复！
```

### 11.5.4 EMIF可编程异步接口 ⭐

**接口时序参数：**

| 参数 | 含义 | 最小值 |
|:-----|:-----|:------:|
| **Setup(建立时间)** | 访问开始到读/写有效之间时间 | 1个ECLKOUT周期 |
| **Strobe(选通时间)** | 读/写有效到无效之间时间 | 1个ECLKOUT周期 |
| **Hold(保持时间)** | 读/写无效到访问结束之间时间 | 0个ECLKOUT周期 |

**时序约束计算：**
```
与FLASH接口时：
  TSetup + TStrobe + THold ≥ TRC min
  TSetup + TStrobe ≥ TAA max
  TStrobe ≥ TOE max
```

**DSP与FLASH连接示例：**
```
DSP                          FLASH
EA[20:1]  ────→ A[19:0]
ED[7:0]   ←───→ D[7:0]
BCE1#     ────→ CE#
BARE#     ────→ OE#
BAWE#     ────→ WE#
```

### 11.5.5 EMIF可编程同步接口

| 特性 | 说明 |
|:-----|:------|
| **连接的存储器** | 同步FIFO、ZBTSRAM、SBSRAM |
| **接口时序** | 可编程（CExSEC寄存器配置） |
| **控制信号** | SOE#、SOE3#、SWE#、SADS/SRE#、ECLKOUT1/2 |
| **读出延迟** | SYNCRL → 最大3个周期 |
| **写入延迟** | SYNCWL → 最大3个周期 |

## 🧠 本章口诀

```
DSP全称有两意：数字信号处理和技术
信号处理核心操，单周期做乘加(MAC)

DSP特点要记住：
  超级哈佛双存储，多路总线并行跑
  流水线填满效率高，硬件乘法是核心
  专用指令复合强，片内集成外设多

三大平台要分清：C2000控制C5000省电
  C6000性能最强，64x定点C64x

C64x结构不简单，八功能单元分AB组
  LM算术SD寻址，M单元专门做乘法
  VLIW一次发八条，单周期内并行算

两级存储L1+L2，L1分数据和程序
  EDMA传输不用CPU，64通道优先级
  EMIF接口好编程，异步同步都能连

FLASH分NOR和NAND：
  NOR随机快可运行，NAND大容量按页访
  FIFO数据流水管，空满异常要注意
```
