## 10.1 🖥️ 嵌入式系统概述

### 10.1.1 什么是嵌入式系统？

> **IEEE定义：** 用于控制、监视或协助设备、机器或工厂运行的装置

**核心定义：**
```
以应用为中心，以计算机技术为基础，软硬件可裁剪，
适应应用系统对功能、可靠性、成本、体积、功耗等严格要求的专用计算机系统
```

**通俗理解：**
```
是一种"完全嵌入到受控设备内部，为特定应用而设计的专用计算机系统"
是相对桌面系统来讲的，凡是带有微处理器的专用软硬件系统都可以称为嵌入式系统
是软件和硬件的综合体，还可以涵盖机械等附属装置
```

### 10.1.2 嵌入式系统的组成

```
嵌入式系统 = 嵌入式硬件 + 嵌入式软件

嵌入式硬件：以芯片、模板、组件、控制器形式存在于设备内部
嵌入式软件：实时多任务操作系统和各种专用软件，一般固化在ROM或闪存中

嵌入式系统软硬兼施，融为一体，成为产品
```

### 10.1.3 嵌入式图像处理系统的三大特点

| 特点 | 含义 | 具体要求 |
|:----|:-----|:---------|
| **"嵌入性"** 🔌 | 嵌入到应用系统中 | 满足"机、电、热"环境要求（接口、体积、功耗、可靠性） |
| **"专用性"** 🎯 | 面向特定应用 | 软硬件专门满足图像处理要求 |
| **"处理系统"** ⚡ | 与应用系统配合 | **图像处理能力是根本** |

## 10.3 🔲 图像处理器

### 处理器类型对比

| 类型 | 全称 | 特点 | 适用场景 |
|:----|:-----|:-----|:---------|
| **DSP** 🔢 | Digital Signal Processor | 适合信号处理算法，编程灵活 | 视频跟踪、目标识别 |
| **FPGA** 🔧 | Field Programmable Gate Array | **细粒度并行**，接口灵活 | 接口控制、并行图像处理 |
| **GPU** 🎮 | Graphic Processing Unit | 大规模并行计算 | 深度学习、图形渲染 |
| **ASIC** 📟 | Application Specified IC | 专用定制，功耗最低 | 量产产品 |
| **SoC** 🧩 | System on Chip | 集成处理器+外设+内存 | 嵌入式主控 |
| **SoPC** 🔬 | System on Programmable Chip | 可编程片上系统 | 灵活定制 |
| **NPU** 🤖 | Neural Processing Unit | **神经网络加速** | AI智能识别 |

### FPGA芯片分类

| 类型 | 编程特性 | 代表厂商 | 特点 |
|:----|:---------|:---------|:------|
| **SRAM型FPGA** | **可重复编程** | Xilinx(AMD)、Altera(Intel) | 大规模、高性能，需外部配置器件 |
| **FLASH型FPGA** | 可重复编程 | Actel(Microsemi) | **上电即运行**，不需配置器 |
| **CPLD** | 可重复编程 | Xilinx、Altera | 小规模逻辑 |
| **反熔丝FPGA** | **一次性编程(OTP)** | Actel | 高可靠应用（航天） |

> **FPGA在图像处理中的应用：**
> 1. 实现接口与控制逻辑 → FPGA是接口"万金油"
> 2. 完成图像处理任务 → 细粒度并行处理"得天独厚"
> 3. FPGA几乎不可缺少！

### 10.4.1 供电电路

**双（多）电源供电：**
```
I/O电压（如3.3V）
内核电压（如1.2V/1.4V）

设计要求：
  ✅ 电压范围满足要求
  ✅ 电流足够（考虑上电启动电流 > 正常工作电流）
  ✅ 内核电压不迟于I/O电压上电
  ✅ 电压爬升速度符合要求
  ✅ 散热设计
```

**两类DC/DC电源对比：**

| 类型 | 原理 | 优点 | 缺点 | 合适场合 |
|:----|:-----|:-----|:-----|:---------|
| **线性稳压(LDO)** 🔋 | 线性调整 | 电路简单、纹波小 | **效率低**、热耗大 | 低压降、低电流 |
| **开关电源(DC/DC)** ⚡ | PWM控制开关 | **效率高**(>95%)、热耗低 | 纹波较大(数十mV) | 高压降、高电流 |

**LDO效率：**
```
效率 η = Vout/Vin × 100%
热耗 = Iout × (Vin - Vout)

例：5V→1.2V，效率仅 1.2/5 = 24%！ → 热耗很大！
```

**多级变电供电：**
```
一级变电        二级变电
输入电压        DC/DC              LDO
(如28V)    →   开关电源     →      I/O电压 (3.3V)
                              →   内核电压1 (1.4V)
                              →   内核电压2 (1.2V)

前级用高效率开关电源 → 降低压差
后级用低纹波线性电源 → 保证供电质量
```

### 10.4.2 时钟电路

**两种时钟产生方式：**

| 方式 | 原理 | 优点 | 缺点 |
|:----|:-----|:-----|:-----|
| **无源晶振** 🎵 | 需要器件内部集成谐振电路 | 成本低 | 需要芯片支持 |
| **有源晶振** 🎵 | 晶振模块直接输出时钟 | 信号好，使用简单 | 成本略高 |

**时钟信号时序指标：**
```
周期       T = 1/f
占空比     高/低电平持续时间比
稳定时间   上电到时钟稳定的时间
沿跳变时间  上升沿/下降沿时间
周期抖动   周期稳定度
```

**时钟驱动/管理：**
```
驱动器件       → 提供多路同相位时钟（如IDT3805）
带PLL的时钟驱动 → 倍频/分频/相位调整（如CY2308）
FPGA时钟管理   → DCM（数字时钟管理器）
```

### 10.4.3 复位电路

**复位类型：**
```
冷复位：电路上电即产生的复位
热复位：通电状态下产生的复位（看门狗复位、按键复位）

一般为低电平复位，对低电平保持时间有要求
```

**复位电路方案对比：**

| 方案 | 原理 | 可靠性 | 建议 |
|:----|:-----|:------|:-----|
| **RC复位** 🛠️ | 电容充电延时 | ❌ 不稳定 | **不建议使用** |
| **上电复位器件** ✅ | 如TPS3307，电压监控 | ✅ 可靠 | **DSP系统建议使用** |
| **看门狗复位** 🐕 | 如MAX706R，监控程序运行 | ✅ 防止跑飞 | 用于高可靠系统 |

**上电复位器件TPS3307功能：**
```
✅ 3路电压监控（上电产生复位）
✅ 供电电压低于门限 → 保持复位
✅ 电压高于门限 → 维持Td(200ms)后释放复位
✅ 支持按键复位输入
```

**看门狗复位MAX706R：**
```
DSP提供周期翻转"喂狗"信号WDI
WDI未在1.6s内翻转 → WDO#有效 → 表示程序"跑飞"了
```

## 10.5 ⏱️ 实时图像处理系统

### 10.5.1 实时性分类

| 类型 | 定义 | 例子 | 要求 |
|:----|:-----|:-----|:-----|
| **硬实时系统** 💀 | 规定时间内必须完成 | 导弹制导、防空预警 | 超时=任务失败 |
| **软实时系统** 😌 | 时间弹性或很长 | 人脸识别门禁、车牌识别 | 超时可容忍 |
| **非实时系统** 🎮 | 无严格时间要求 | 消费类产品 | 越快越好 |

### 10.5.2 实时性时序分析

```
输入流                   处理系统                   输出流
I(k,l) ──────→         M(·)          ──────→    O(m,n)
                    ╭─────────╮
Ti(输入周期)        │ Tp处理时间 │               To(输出周期)
                    ╰─────────╯

时序关系：
  Tp是动态变化的
  若To ≤ Ti，ΔT = Ti - To：系统等待新输入的时间 → 实时系统
  输出结果存在几帧处理延迟，至少以Ti为间隔周期输出 → 准实时系统
```

### 10.5.3 实时图像处理面临挑战

```
挑战一：图像数据率骤增
  📈 成像传感器帧频不断提高 → 处理周期不断减小
  📈 成像分辨率不断提高 → 图像数据量不断增加

挑战二：信息处理日趋复杂
  📈 信息获取手段多样化
  📈 信息处理流程复杂化 → 系统规模急剧增大

挑战三：系统需求提升
  📈 I/O吞吐率增加
  📈 存储容量增大
  📈 计算资源增多
```

## 10.6 🔄 串行处理的瓶颈

### 冯·诺依曼瓶颈

```
传统CPU架构的问题：
  ① 取指/译码/取数/执行 → 串行处理
  ② 所有工作都交由ALU完成
  ③ 大部分时间耗在访存
  ④ 统一寻址访存，数据频繁往返于ALU和存储器

后果：
  ❌ 复杂计算中，数据频繁在ALU和存储器间传递
  ❌ 访存带宽最终制约算法速度
```

### 传统优化方向

```
为了突破瓶颈，传统CPU架构不断进化：

  📈 增加主频（但功耗墙限制了频率提升）
  📈 增加字长：4→8→16→32→64 bit
  📈 高速缓存（Cache）：加速数据/指令访存
  📈 哈佛结构：指令和数据分离，加倍存储带宽
  📈 流水线：取指/译码/执行叠加重叠
  📈 精简指令集(RISC)：更高效利用资源
  📈 多核架构：多核并行处理
```

## 10.7 ⚡ 并行处理架构 ⭐⭐

### 为什么用并行处理？

```
关键部分专设硬件：
  ✅ 硬件本质上是并行的
  ✅ 使用局部寄存器存储中间结果

增大单次计算能力：
  ✅ 每个处理器有各自局部存储器
  ✅ 多处理器并行工作

FPGA成为常用并行计算平台 → FPGA资源足够时，可以实现高效并行
```

### 10.7.1 时间并行（操作级流水线）

```
算法由一系列顺序操作组成 → 每个操作用一个单独的处理器

输入 → Processor 1 → Processor 2 → Processor 3 → 输出
       (Operation 1) (Operation 2) (Operation 3)

优点：各操作并行执行，吞吐率大幅提升

缺点：
  ❌ 不适合迭代算法
  ❌ 受制于操作间数据传递带宽
```

### 10.7.2 空间并行（数据级并行）

```
多数操作在大量像素上执行相同功能 → 将图像分割到多个处理器

图像分割方式：
  ┌──────────────────────┐
  │  行分割  │  列分割  │  块分割  │
  │  Row     │  Column  │  Block   │
  └──────────────────────┘

架构：SIMD（单指令多数据流）
要求：操作只需局部区域数据时效率最高
```

### 10.7.3 大规模并行

```
极致空间并行：每个像素一个独立的处理器！
  ✅ 某些算法效率极高
  ❌ 资源消耗巨大（随图像尺寸不扩展）
  ❌ 仅适用于小图像
  ❌ 图像传入/传出处理器阵列需大量时间
```

### 10.7.4 流处理 ⭐

> **将图像按像素扫描顺序串行化，把空间并行转化为时间并行**

```
图像 → 像素流 → Pipeline操作1 → Pipeline操作2 → ... → 输出

      Row 0  Row 1  Row 2  Row 3  ...  Row N
时间 →  →  →  →  →  →  →  →  →  →  →  →  →  →

特点：
  ✅ 每个像素顺序处理 → 不需要并行数据访问
  ✅ 适合流水线架构
  ✅ 数据在流水线的操作之间以流形式传递
  ✅ 操作内部可实现细粒度流水线
```
