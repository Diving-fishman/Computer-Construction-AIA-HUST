## 🎯 为什么学这门课？

### 理由一：国家需要 🇨🇳

```
"缺芯少魂" → 自主CPU(龙芯/飞腾)、自主OS(麒麟/鸿蒙)、自主编译器
```

> 龙芯、飞腾、华为鲲鹏... 这些都是我们国家自主研发的CPU
> 要设计CPU，必须先懂计算机组成原理！

### 理由二：考研必考 📝

计算机考研**408专业课**中，"计算机组成原理"占**45分**，是四门课中分值最高的之一！

### 理由三：构建"软硬协同"的系统观 🔗

```
只会写软件 → 不知道程序在硬件上怎么跑的 → 遇到性能问题只能干瞪眼
懂硬件原理 → 能写出更高效的代码 → 能理解为什么这样写更快
```

### 理由四：重要的"承上启下"课程

```
┌─────────────────────────────────┐
│   高级语言程序设计 / 数据结构      │ ← 前置基础（你会写代码了）
├─────────────────────────────────┤
│   🔷 计算机组成原理 ← 你在这里  │ ← 搞清楚代码怎么在硬件上跑
├─────────────────────────────────┤
│   操作系统 / 编译原理 / 体系结构   │ ← 后续课程（有了基础再深入）
└─────────────────────────────────┘
```

## 🧩 有趣的例子

> 这些例子都是第一节课老师就会讲的，非常有意思！

### 例子1：浮点数不精确 🤯

```c
#include <stdio.h>
main() {
    double a, b, c;
    int d;
    b = 3.3;
    c = 1.1;
    a = b / c;        // 你以为 a = 3.0
    d = b / c;        // 你以为 d = 3
    if (3.0 != a)
        printf("Really? 3.0 != a");  // 😱 居然输出了这句话！
}
```

**为什么？🤔**

```
3.3 的二进制：11.010011001100110011... 无限循环！
1.1 的二进制：1.00011001100110011... 无限循环！

就像 1/3 = 0.333... 在十进制中无法精确表示一样
很多十进制小数在二进制中是无限循环的 → 浮点数只能存近似值
```

**启示：** 永远不要直接用 `==` 比较浮点数！应该用 `|a - b| < ε` 来判断

### 例子2：U盘为什么要"安全删除"？💾

```
你以为的拷贝流程：
你拖拽文件到U盘 → 进度条走完 → 文件已复制完成 ✅

实际上的拷贝流程：
你拖拽文件到U盘 → 进度条走完 → 文件写到了"内存缓冲区"
                              → 系统后台慢慢把数据从内存写到U盘
                              → 此时立刻拔U盘 → 数据还在缓冲区，没写到U盘！❌
```

**启示：**
- "安全删除" = 确保缓冲区里的脏数据都写入了U盘
- 如果只是从U盘**读**数据（U盘→电脑），不涉及写操作，可以直接拔 ✅
- 如果是**写**数据到U盘，最好等一会再拔，或者用"安全删除"

### 例子3：数组遍历顺序影响性能 🏎️

```c
// 方法A：按行遍历 → 超级快 🏆
for (i = 0; i < 10000; i++)
    for (j = 0; j < 10000; j++)
        sum += a[i][j];

// 方法B：按列遍历 → 超级慢 🐢
for (j = 0; j < 10000; j++)
    for (i = 0; i < 10000; i++)
        sum += a[i][j];
```

**两种方式结果一样，但速度可能差几十倍！为什么？**

```
C语言中二维数组的存储方式（行优先）：
  a[0][0] a[0][1] a[0][2] ... a[1][0] a[1][1] ...
   ← 一行挨着一行 →

方法A：按顺序访问 a[0][0], a[0][1], a[0][2]...
       → 连续的内存地址 → Cache预读有效 → 命中率高

方法B：跳着访问 a[0][0], a[1][0], a[2][0]...
       → 相隔很远的地址 → Cache预读无效 → 命中率低
```

**这就是"CPU Cache的空间局部性原理"！** 我们会在第4章详细学

## 💡 怎么学这门课？

### 三种学习视角

| 视角 | 核心问题 | 类比 |
|:----|:---------|:-----|
| 🔨 **构造观** | "怎么设计一个计算机？" | 像搭积木一样，把各部件搭起来 |
| 🔗 **系统观** | "各部件怎么协同工作？" | 像指挥乐队，每个乐器配合 |
| 🏭 **工程观** | "怎么做的更好更快更省？" | 像工程师做优化 |

### 学习建议 📝

```
1. 多动手实验 🖥️
   - 使用 Logisim 平台搭建电路
   - 纸上画数据通路图
   - 做课后习题

2. 多思考"为什么" ❓
   - 为什么用补码不用原码？
   - 为什么要有Cache？
   - 为什么指令要分成不同格式？

3. 建立系统观 🔗
   - 把各个章节联系起来看
   - 理解一个改动如何影响整体

4. 善用类比 🧩
   - 把抽象概念和生活联系起来
   - 比如：Cache = 桌面，主存 = 书包
```
