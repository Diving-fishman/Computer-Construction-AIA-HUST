## 5.1 🌐 指令系统概述

### 什么是指令系统？

> **指令系统 (Instruction Set Architecture, ISA)** = CPU能识别和执行的**全部指令的集合**

**打个比方：**
```
你和外国人交流 → 你们需要共同的语言 → 中文/英文
程序员和CPU交流 → 需要共同的语言 → 指令系统(ISA)
```

### 常见的ISA

| ISA | 发布年份 | 设计哲学 | 主要应用 |
|:----|:-------:|:---------|:---------|
| **x86** 💻 | 1978 | CISC（复杂） | PC、服务器（Intel/AMD） |
| **MIPS** 📚 | 1981 | RISC（精简） | 教学、嵌入式 |
| **ARM** 📱 | 1991 | RISC | 手机、平板（90%+移动市场） |
| **RISC-V** 🔓 | 2016 | RISC、**开源免费** | 新兴之星 |
| **LoongArch** 🐉 | 2020 | RISC、**中国自主** | 龙芯CPU |

### CPU设计四大原则 ⭐

> 这四个原则贯穿整个CPU设计，一定要记住！

```
① 简单来自规整 🎯
   → 指令格式统一，设计就简单
   → 例：MIPS所有指令都是32位

② 越小越快 📏
   → 面积越小，电路延迟越短
   → 例：MIPS寄存器只用5位(32个)，不设计成6位(64个)

③ 加快经常性事件 ⚡
   → 常用操作设计得特别快
   → 例：加法频率远高于除法 → 加法器优化得好

④ 好设计需要折中 ⚖️
   → 没有完美的方案，只有合适的权衡
   → 例：指令长度固定(Vs)vs指令功能丰富
```

## 5.2 📝 指令格式

### 指令的组成

一条指令包含两部分：

```
┌──────────────────────────────────┐
│        操作码 (OP)               │
│  告诉CPU"做什么"                 │
│  例：加法、减法、跳转...          │
├──────────────────────────────────┤
│        地址码 (AC)               │
│  告诉CPU"对谁做"                 │
│  例：寄存器编号、内存地址...       │
├──────────────────────────────────┤
│        寻址方式 (可选)            │
│  告诉CPU"怎么找到操作数"          │
└──────────────────────────────────┘
```

### 指令格式分类（按操作数数量）

| 类型 | 格式 | 含义 | 示例 |
|:----|:-----|:------|:------|
| **三地址** 📐 | `OP A1, A2, A3` | A2 OP A3 → 结果存入A1 | `ADD R1, R2, R3` # R1=R2+R3 |
| **二地址** 📐 | `OP A1, A2` | A1 OP A2 → 结果存入A1 | `ADD R1, R2` # R1=R1+R2 |
| **单地址** 📏 | `OP A1` | AC OP A1 → AC | `ADD 100` # 累加器+=100 |
| **零地址** 🔵 | `OP` | 不需要操作数 | `HALT` # 停机 |

### 扩展操作码技术 ⭐⭐

> **问题：** 指令长度有限，但想支持更多指令怎么办？
> **答案：** 操作码长度可变！长的操作码从短操作码的"剩余"状态中扩展出来

**例子：16位指令格式**

```
假设16位指令，地址码各6位：

① 双操作数指令（用4位操作码）
   OP(4位) | A1(6位) | A2(6位)
   还剩 2⁴ - 15 = 1种状态可以扩展

② 单操作数指令（用10位操作码）
   从上面1种状态扩展出6位 → 2⁶ = 64种
   用了34种 → 还剩 64-34 = 30种

③ 零操作数指令（用16位操作码）
   从上面30种扩展出6位 → 30 × 2⁶ = 1920种
```

> **理解关键：** 扩展操作码就像"抽屉原理"
> 先把常用指令分配短码（如ADD/SUB用4位）
> 不常用指令分长码（如专门的系统指令用16位）
> 短码剩下的编码空间给长码用

## 5.3 🎯 寻址方式 ⭐⭐

### 什么是寻址方式？

> **寻址方式 = 告诉CPU怎么找到操作数的"地图"🗺️**

举例子：`ADD R1, [地址]`
```
这个"地址"怎么理解？
- 就是R1本身？→ 寄存器寻址
- 是R1里存的数据？→ 寄存器间接寻址
- 是个具体的数字？→ 立即数寻址
- 是个内存地址？→ 直接寻址
- PC+偏移量？→ 相对寻址
```

### 七种常见寻址方式

假设指令 `OP A`，A是"形式地址"，E是"有效地址"（真正的内存地址）

| 寻址方式 | 有效地址 E | 访存次数 | 通俗理解 | 范围 |
|:---------|:----------:|:--------:|:---------|:----:|
| **立即寻址** 🔢 | 操作数=A本身 | **0次** | 指令里直接写数字 | 立即数大小 |
| **寄存器寻址** 🗂️ | 操作数在R中 | **0次** | 操作数在寄存器里 | 寄存器编号 |
| **直接寻址** 🎯 | E=A | **1次** | A就是内存地址 | 整个内存 |
| **间接寻址** 🔍 | E=(A) | **2次** | A存的是地址的地址 | 整个内存 |
| **寄存器间接** 📎 | E=(R) | **1次** | 寄存器里存的是地址 | 整个内存 |
| **相对寻址** 📍 | E=PC+A | **1次** | 相对于当前指令的位置 | PC上下范围 |
| **变址寻址** 📏 | E=R+A | **1次** | 基址+偏移量 | 整个内存 |

**每种方式的直观理解：**

```
立即寻址：   ADD R1, #100    ← 直接把100这个数字加给R1
                                    (100就在指令里)

寄存器寻址： ADD R1, R2      ← R1 = R1 + R2
                                    (R2里存的是数据)

直接寻址：   LW R1, 1000     ← 从内存地址1000取数据
                                    (地址1000在指令里)

寄存器间接： LW R1, (R2)     ← R2里存的是地址，去那个地址取数据
                                    (R2=2000 → 去地址2000取)

相对寻址：   BEQ R1,R2,100   ← 如果相等，跳到PC+100的位置
                                    (用于条件跳转)

变址寻址：   LW R1, 100(R2)  ← 去地址 R2+100 取数据
                                    (用于数组访问，R2=基址，100=偏移)
```

### 堆栈寻址

| 类型 | 原理 | 优点 | 缺点 |
|:----|:-----|:-----|:------|
| **硬件堆栈** 🔲 | 寄存器串联 | 速度快 | 容量小（寄存器个数有限） |
| **软件堆栈** 💾 | 内存区域 + SP指针 | **容量大**、灵活 | 速度较慢 |

> **软件堆栈是主流！** 函数调用、参数传递都用它
> SP(Stack Pointer)指向栈顶，push=SP减，pop=SP加

## 5.4 🏷️ 指令类型

### 六大类型

```
① 数据传送指令 🚚
   → 数据在寄存器/内存间搬来搬去
   → MOV, LW(Load Word), SW(Store Word), LI(加载立即数)

② 算术运算指令 ➕
   → 做数学计算
   → ADD, SUB, MUL, DIV, ADDI(加立即数)

③ 逻辑运算指令 🔲
   → 做位运算
   → AND, OR, XOR, NOR, SLL(左移), SRL(右移)

④ 程序控制指令 🎮
   → 改变程序的执行顺序
   → JMP(无条件跳转), BEQ(相等则跳), JAL(调用函数), JR(函数返回)

⑤ 输入输出指令 📨
   → 和外设交换数据
   → IN, OUT (x86中的I/O指令)

⑥ 系统控制指令 ⚙️
   → 操作系统级别操作
   → SYSCALL, BREAK
```

## 5.5 🏗️ MIPS指令集详解 ⭐⭐⭐

> MIPS是**典型的RISC架构**，所有指令固定为32位，
> 只有**三种格式**，非常规整，便于学习！

### MIPS寄存器

MIPS有**32个通用寄存器**，每个32位，用编号$0~$31表示

| 编号 | 别名 | 用途 | 谁负责保存 |
|:----:|:----:|:------|:----------:|
| `$0` | `$zero` | 恒为0（硬件实现） | — |
| `$1` | `$at` | 汇编器保留 | — |
| `$2-$3` | `$v0-$v1` | 函数返回值 | 被调用者 |
| `$4-$7` | `$a0-$a3` | 函数参数 | 被调用者 |
| `$8-$15` | `$t0-$t7` | 临时变量 | **调用者** |
| `$16-$23` | `$s0-$s7` | 保存变量 | **被调用者** |
| `$24-$25` | `$t8-$t9` | 临时变量 | 调用者 |
| `$28` | `$gp` | 全局指针 | — |
| `$29` | `$sp` | 栈指针 | — |
| `$30` | `$fp` | 帧指针 | — |
| `$31` | `$ra` | 返回地址 | — |

> **记忆技巧：** 
> - `$t` = temporary（临时，用完不管）
> - `$s` = saved（要保存，用前存起来用完恢复）
> - `$a` = argument（参数）
> - `$v` = value（返回值）

### MIPS的三种指令格式

#### ① R型 (Register) — 寄存器操作

```
       6位     5位     5位     5位     5位     6位
   ┌────────┬──────┬──────┬──────┬──────┬────────┐
   │  OP   │  Rs  │  Rt  │  Rd  │shamt │ funct  │
   │ 操作码 │源操作数│源操作数│目的   │移位量│功能码  │
   │  000000│      │      │      │      │        │
   └────────┴──────┴──────┴──────┴──────┴────────┘
```

**R型指令特点：**
```
所有操作数都在寄存器中
OP=000000时，具体操作由funct决定
shamt用于移位指令(如sll)
```

**R型指令示例：**
```
add $1, $2, $3    → 机器码: 000000 00010 00011 00001 00000 100000
                      OP=0  Rs=$2  Rt=$3  Rd=$1  shamt=0  funct=32

sub $1, $2, $3    → funct=34 (100010)
sll $1, $2, 10    → shamt=10, funct=0
jr $31            → 跳转到 $31 中的地址
```

#### ② I型 (Immediate) — 立即数/数据传送

```
       6位     5位     5位         16位
   ┌────────┬──────┬──────┬──────────────────────┐
   │  OP   │  Rs  │  Rt  │       立即数          │
   │ 操作码 │基址   │源/目的│    立即数/偏移量     │
   └────────┴──────┴──────┴──────────────────────┘
```

**I型指令特点：**
```
有一个16位立即数（可表示 -32768~32767）
用于：加立即数、访存、条件分支
```

**I型指令示例：**
```
① 加立即数
   addi $1, $2, 100   → $1 = $2 + 100
   机器码: 001000 00010 00001 0000000001100100
           OP=8    Rs=$2  Rt=$1  立即数=100

② 取字（从内存加载）
   lw $1, 100($2)     → $1 = Mem[$2 + 100]
   机器码: 100011 00010 00001 0000000001100100
           OP=35   Rs=$2  Rt=$1  偏移量=100

③ 存字（存入内存）
   sw $1, 100($2)     → Mem[$2 + 100] = $1
   机器码: 101011 00010 00001 0000000001100100
           OP=43   Rs=$2  Rt=$1  偏移量=100

④ 相等则跳转
   beq $1, $2, label → 如果$1==$2，跳到label
   机器码: 000100 00001 00010 偏移量(相对PC)
           OP=4    Rs=$1  Rt=$2  相对PC的偏移
```

#### ③ J型 (Jump) — 跳转

```
       6位             26位
   ┌────────┬──────────────────────────────┐
   │  OP   │             地址              │
   │ 操作码 │          跳转目标             │
   │  2或3  │                              │
   └────────┴──────────────────────────────┘
```

**J型指令示例：**
```
j label      → 无条件跳转到label
jal func     → $ra = PC+4; 跳转到func（调用函数）
```

### MIPS指令编码速查表

| 指令 | 格式 | OP码 | funct码 | 含义 |
|:----|:----|:----:|:-------:|:------|
| **add** | R | 0 | 32 | Rd = Rs + Rt |
| **sub** | R | 0 | 34 | Rd = Rs - Rt |
| **and** | R | 0 | 36 | Rd = Rs & Rt |
| **or** | R | 0 | 37 | Rd = Rs \| Rt |
| **slt** | R | 0 | 42 | Rd = (Rs < Rt) ? 1:0 |
| **sll** | R | 0 | 0 | Rd = Rt << shamt |
| **jr** | R | 0 | 8 | 跳转到 Rs |
| **addi** | I | 8 | — | Rt = Rs + imm |
| **lw** | I | 35 | — | Rt = Mem[Rs + imm] |
| **sw** | I | 43 | — | Mem[Rs + imm] = Rt |
| **beq** | I | 4 | — | if(Rs==Rt) PC += imm |
| **bne** | I | 5 | — | if(Rs!=Rt) PC += imm |
| **j** | J | 2 | — | PC = target |
| **jal** | J | 3 | — | $ra=PC+4; PC=target |

### MIPS函数调用约定

```
调用函数：
jal Func     → $ra = 返回地址(当前PC+4)
                PC = Func 的函数入口
                
函数内部：
addi $sp, $sp, -4    → 栈顶下移
sw   $ra, 0($sp)     → 保存返回地址到栈上
...                   → 函数体代码...
lw   $ra, 0($sp)     → 恢复返回地址
addi $sp, $sp, 4     → 栈顶上移
jr   $ra             → 返回调用者
```

> **函数调用的"舞蹈"** 💃：
> 1. 调用者用 `jal` 跳转 → 硬件自动把返回地址存到 `$ra`
> 2. 被调用者在栈上保存 `$ra`
> 3. 函数结束后从栈恢复 `$ra`，用 `jr $ra` 返回

### MIPS汇编示例：数组求和

```asm
# int sum(int *arr, int n)
# $a0 = arr, $a1 = n
# 返回值在 $v0

sum:
    addi $v0, $zero, 0    # sum = 0
    addi $t0, $zero, 0    # i = 0
    
loop:
    beq  $t0, $a1, done   # if i == n, break
    sll  $t1, $t0, 2      # t1 = i * 4 (字节偏移)
    add  $t1, $a0, $t1    # t1 = &arr[i]
    lw   $t2, 0($t1)      # t2 = arr[i]
    add  $v0, $v0, $t2    # sum += arr[i]
    addi $t0, $t0, 1      # i++
    j    loop
    
done:
    jr   $ra              # return sum
```

## 5.6 🆚 CISC vs RISC ⭐

### 两种设计哲学

| 对比项 | **CISC** 🏢 (x86) | **RISC** 🏠 (MIPS/ARM/RISC-V) |
|:-------|:------------------|:-----------------------------|
| **指令数量** | 多（x86有上千条） | 少（几十到几百条） |
| **指令长度** | 变长（1~15字节） | **定长**（32位） |
| **寻址方式** | 复杂多样 | 简单几种 |
| **寄存器数量** | 少（x86:8个通用） | **多**（MIPS:32个） |
| **访存方式** | 运算指令可访存 | **只有Load/Store可访存** |
| **控制器实现** | 微程序控制（复杂） | **硬布线控制**（简单快速） |
| **是否有标志位** | 有 (EFLAGS) | 无 |
| **代表** | Intel x86, AMD | MIPS, ARM, RISC-V |

### CISC的典型特点

> **"一条指令干很多事情"**

```
x86示例：MOV指令可以：
- 寄存器到寄存器
- 寄存器到内存
- 内存到寄存器
- 立即数到寄存器
- 立即数到内存
- 各种组合...

一条指令 = 多种功能 → 指令译码复杂 → 控制器复杂
```

### RISC的典型特点

> **"一条指令只干一件简单的事情"**

```
MIPS的规则：
- 所有指令等长（32位）→ 取指简单
- 只有Load/Store访存 → 容易区分
- 3种格式 → 译码简单
- 大量寄存器 → 减少访存
```

### 现代趋势：互相融合 🔀

```
现代x86 CPU内部：
  外部指令(x86 CISC) → 译码器 → 内部微指令(RISC-like) → 执行

这就是为什么：
  x86功耗高（译码复杂）
  x86性能不差（内部其实是RISC核心）

RISC也在增加指令：
  ARM的NEON -> SIMD指令变复杂
  RISC-V允许可选扩展
```

## ⭐ MIPS必会指令

```
add $1,$2,$3    # R型：$1=$2+$3
sub $1,$2,$3    # R型：$1=$2-$3
addi $1,$2,100  # I型：$1=$2+100
lw $1,100($2)   # I型：从内存读数据
sw $1,100($2)   # I型：写数据到内存
beq $1,$2,label # I型：相等则跳
bne $1,$2,label # I型：不等则跳
j label         # J型：无条件跳转
jal label       # J型：调用函数（跳+存返回地址）
jr $ra          # R型：函数返回
sll $1,$2,10    # R型：左移
slt $1,$2,$3    # R型：小于则置1
```
