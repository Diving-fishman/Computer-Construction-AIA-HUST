## 1.1 ⏳ 计算机发展历程

### 1.1.1 远古计算工具 🦴

人类对"算"的追求从古至今从未停止：

| 时代 | 工具 | 什么原理？ | 意义 |
|:----:|:----:|:-----------|:-----|
| **春秋战国** | 算筹 🥢 | 用一捆小棍子摆出数字 | 最早的十进制计算工具 |
| **西周** | 算盘 🧮 | 上珠下珠代表5和1 | "第五大发明"，至今仍用！ |
| **1621年** | 计算尺 📏 | 用对数原理做乘除 | 工程师的300年伙伴 |
| **1642年** | Pascal加法机 ⚙️ | 齿轮传动进位 | **第一台机械计算机** |
| **1673年** | 莱布尼兹乘法机 | 步进轮实现四则运算 | 能乘除了！ |

> **算盘的原理**：上珠一个=5，下珠一个=1。每一档代表一个数位。
> 口诀"三下五除二"就是在说：加3 = 拨下上珠(5) 同时 拨去下珠2个(5-2=3)

### 1.1.2 "程序控制"思想的诞生 💡

这是计算机史上**最重要的思想飞跃**！

| 年份 | 人物 | 贡献 | 通俗理解 |
|:----:|:----|:-----|:---------|
| **1725** | Bouchon | 穿孔纸带控制织布机 🎭 | 纸带上的洞=指令 |
| **1805** | Jacquard | 提花机用卡片"编程" | 换一张卡片=换一种花布图案 |
| **1821** | Babbage | 设计**差分机** + **分析机** | 现代计算机的"精神始祖" |
| **1842** | **Ada Lovelace** 👩‍💻 | 写了第一个"程序"（计算伯努利数） | **世界上第一位程序员！** |

> **💡 关键转折：** 提花机用穿孔卡片控制花纹 → 启发Babbage：能不能用卡片控制计算？
> → 从此，"程序控制"的思想诞生了！

## 1.2 🧠 计算机系统组成

### 1.2.1 冯·诺依曼结构 ⭐⭐⭐

> **冯·诺依曼** 在1945年提出了"存储程序"的概念，至今几乎所有计算机都遵循这一思想！

#### 核心思想：两句话

```
① 存储程序：把程序（指令序列）和数据一起存放在存储器中
② 程序控制：从存储器中逐条取出指令、自动执行
```

**打个比方 📝：**
```
就像你做菜：
① 把菜谱(程序) 和 食材(数据) 都放在案板上(存储器)
② 按照菜谱一步一步做(逐条执行指令)
   → 如果你换一本菜谱，就做不同的菜（程序=可更换！）
```

#### 五大部件

```
                    ┌─────── 存储器(Memory) ───────┐
                    │    存放程序和数据              │
                    └──┬───────────────────────┬───┘
                       │                      │
               ┌───────┘                      └───────┐
               ▼                                      ▼
        ┌────────────┐                        ┌────────────┐
        │  输入设备   │                        │  输出设备   │
        │  (Input)   │                        │  (Output)   │
        │  键盘/鼠标  │                        │  显示器/打印 │
        └────────────┘                        └────────────┘
               ▲                                      ▲
               │                                      │
               └─────── CPU (中央处理器) ─────────────┘
                       │                │
                   ┌───┴───┐       ┌───┴───┐
                   │ 运算器 │       │ 控制器 │
                   │  (ALU) │       │  (CU)  │
                   │ 做计算 │       │ 发号施令│
                   └───────┘       └───────┘
```

| 部件 | 功能 | 类比 🏫 |
|:-----|:-----|:---------|
| **运算器(ALU)** 🧮 | 算术运算(+ - × /) + 逻辑运算(与或非) | 学生，负责做题 |
| **控制器(CU)** 🎮 | 取指令→分析指令→发控制信号 | 班长，负责指挥谁干啥 |
| **存储器** 📦 | 存程序和数据 | 课本和笔记本 |
| **输入设备** ⌨️ | 把外部信息变成计算机能懂的形式 | 眼睛耳朵，接收信息 |
| **输出设备** 🖥️ | 把计算机结果变成人能懂的形式 | 嘴巴，说给人听 |

#### 冯·诺依曼结构的三个要点

```
1️⃣ 计算机由五大部件组成 ✅
2️⃣ 指令和数据以同等地位存放在存储器中 ✅
3️⃣ 指令和数据用二进制表示 ✅
```

### 1.2.2 硬件 vs 软件 vs 固件

| 概念 | 定义 | 特点 | 例子 |
|:----|:-----|:-----|:-----|
| **硬件** 🪨 | 物理实体 | 看得见摸得着 | CPU、内存条、硬盘 |
| **软件** 🧠 | 程序+数据集合 | 是一串二进制信息 | Windows、QQ、Chrome |
| **固件** 🔒 | 固化在ROM里的软件 | 介于软硬之间 | BIOS/UEFI |

> **软硬件逻辑等效性：** 一个功能既可以用硬件实现（快、贵、不易改），也可以用软件实现（慢、便宜、易改）
> 比如：乘法可以用硬件乘法器(快)，也可以用软件循环累加(慢)

## 1.3 📊 计算机系统层次结构

### 从用户到硬件的七层

```
用户(你) 
    ↓     👆 越来越抽象
┌──────────────────────────────┐
│     应用程序层 🎮             │  ← 你在用的软件(QQ/浏览器)
├──────────────────────────────┤
│     高级语言层 📝             │  ← C/C++/Java/Python代码
├──────────────────────────────┤
│     汇编语言层 🛠️             │  ← MOV AX, BX 这种
├──────────────────────────────┤
│     操作系统层 ⚙️             │  ← Windows/Linux内核
├──────────────────────────────┤
│  指令集架构(ISA) 📋           │  ← CPU理解的"母语"
├──────────────────────────────┤
│   微架构/硬件逻辑层 🔌        │  ← 晶体管、门电路
└──────────────────────────────┘
    ↓     👇 越来越底层
硅原子
```

**每一层之间的关系：**
```
上层 = "依赖"下层提供服务
下层 = 对上层"透明"(上层不用关心下层怎么实现的)

就像：
你打开浏览器(应用层) → 不用管操作系统怎么管理内存
你写C语言(高级语言) → 不用管汇编指令长什么样
```

### 程序和指令的执行过程

```
高级语言(C):     a = b + c;
                    ↓ 编译(编译器)
汇编语言:       LW   R1, b     ; 从内存加载b到寄存器
                LW   R2, c     ; 从内存加载c到寄存器
                ADD  R3, R1, R2; R3 = R1 + R2
                SW   a, R3     ; 把结果存回内存
                    ↓ 汇编(汇编器)
机器语言:       100011 00001 00010 ......
                    ↓ 加载到内存
CPU执行:       取指 → 译码 → 执行 → 访存 → 写回
```

## 1.4 📏 计算机系统性能评价 ⭐⭐⭐

### 1.4.1 核心性能指标

#### ① 时钟周期和主频 ⏰

| 概念 | 定义 | 关系 |
|:----|:-----|:-----|
| **主频(f)** 🎵 | CPU每秒的时钟数 | 单位：Hz (赫兹) |
| **时钟周期(T)** ⏱️ | 每个时钟脉冲的时间长度 | T = 1/f |

```
例：主频 2GHz
时钟周期 = 1 / (2×10⁹) = 0.5 × 10⁻⁹ 秒 = 0.5 纳秒(ns)
```

> **通俗理解：** 主频就像心跳频率，时钟周期就是每次心跳的时间

#### ② CPI (Cycles Per Instruction) ⭐

> **CPI = 执行每条指令需要的平均时钟周期数**

```
CPI = 总时钟周期数 / 总指令条数

或者按比例计算：
CPI = Σ(各类指令的CPI × 该类指令占比)
```

**不同指令的CPI不同：**
```
指令类型         CPI
算数运算(ADD) :   1    ← 最快
Load/Store    :   2    ← 需要访问内存
条件转移(BEQ) :   3~4  ← 可能分支预测失败
浮点运算      :   >10  ← 复杂计算
```

#### ③ MIPS (Million Instructions Per Second)

> **MIPS = 每秒能执行多少百万条指令**

```
MIPS = f / (CPI × 10⁶)

f = 主频(Hz)
CPI = 平均每条指令周期数
```

#### ④ MFLOPS

> **MFLOPS = 每秒能执行多少百万次浮点运算**

用于科学计算场景（天气预报、物理模拟等）

### 1.4.2 CPU性能公式 ⭐⭐⭐

```
CPU执行时间 = 指令条数 × CPI × 时钟周期(T)
           = 指令条数 × CPI / 主频(f)
           = 指令条数 / (MIPS × 10⁶)
```

**更完整的表达：**
```
CPU时间 = 用户CPU时间 + 系统CPU时间
用户CPU时间 = 程序在用户态执行的时间
系统CPU时间 = OS为程序服务的时间(如I/O)
```

### 1.4.3 影响性能的因素

| 因素 | 影响指令数 | 影响CPI | 影响主频 |
|:----|:---------:|:-------:|:--------:|
| **算法** 🧠 | ✅ | ✅ | |
| **编程语言** 📝 | ✅ | ✅ | |
| **编译器** 🔄 | ✅ | ✅ | |
| **指令集架构** 📋 | ✅ | ✅ | ✅ |

### 1.4.4 经典例题 ⭐

**例题：** CPU主频1GHz，一个程序有2×10⁵条指令

| 指令类型 | CPI | 占比 |
|:---------|:---:|:----:|
| 算术逻辑 | 1 | 60% |
| Load/Store | 2 | 18% |
| 转移 | 4 | 12% |
| Cache缺失访存 | 8 | 10% |

**求解：**

```
① 平均CPI = 1×60% + 2×18% + 4×12% + 8×10%
           = 0.6 + 0.36 + 0.48 + 0.8
           = 2.24

② MIPS = f / (CPI × 10⁶)
        = 10⁹ / (2.24 × 10⁶)
        = 446.4

③ CPU时间 = 指令条数 × CPI × T
           = 2×10⁵ × 2.24 × 10⁻⁹
           = 4.48 × 10⁻⁴ 秒
           = 0.448 毫秒 ✅
```

### 1.4.5 Amdahl定律（阿姆达尔定律）

> **对系统的某部分加速时，整体性能提升受限于该部分所占的比例**

```
加速比 = 1 / [(1 - 可改进比例) + 可改进比例 / 改进倍数]

例：某程序80%的部分可以并行加速10倍
加速比 = 1 / [(1-0.8) + 0.8/10] = 1/(0.2 + 0.08) = 1/0.28 ≈ 3.57倍
```

> **启示：** 不要只优化一小部分，要找到真正的瓶颈！

## 🧮 计算机中常用的计量单位

### 时间单位

```
1s = 1000 ms (毫秒)
1ms = 1000 μs (微秒)
1μs = 1000 ns (纳秒)
1ns = 1000 ps (皮秒)
```

**直观感受：**
```
人类眨一次眼    ≈ 300ms = 300,000,000ns
CPU执行一条指令 ≈ 0.5ns (主频2GHz)
差距 ≈ 6亿倍！
```

### 存储容量

| 单位 | 字节数 | 通俗理解 |
|:----|:-----:|:---------|
| 1 KB | 2¹⁰ = 1024 B | 一篇短文 |
| 1 MB | 2²⁰ ≈ 10⁶ B | 一张照片 |
| 1 GB | 2³⁰ ≈ 10⁹ B | 一部电影 |
| 1 TB | 2⁴⁰ ≈ 10¹² B | 一台电脑硬盘 |
| 1 PB | 2⁵⁰ ≈ 10¹⁵ B | 大型数据中心 |

### 频率单位

```
1 KHz = 10³ Hz
1 MHz = 10⁶ Hz  
1 GHz = 10⁹ Hz
```

## ⭐ 必背公式

| 公式 | 含义 |
|:----|:------|
| `T = 1/f` | 时钟周期 = 1/主频 |
| `CPI = 总时钟周期数 / 指令总数` | 平均每指令周期数 |
| `CPU时间 = 指令数 × CPI × T` | **CPU性能核心公式** |
| `MIPS = f / (CPI × 10⁶)` | 每秒百万条指令 |
| `加速比 = 1/[(1-α) + α/k]` | Amdahl定律 |
