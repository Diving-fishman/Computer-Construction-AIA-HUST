## 8.1 🌉 总线概述

### 8.1.1 什么是总线？

> **总线 = 一组能为多个部件服务的公共信息传输线路**

**为什么要用总线？**
```
点对点连接（没有总线）：
  每个设备之间单独连线
  
  CPU───内存      ← CPU到内存一根线
  CPU───打印机    ← CPU到打印机一根线
  CPU───键盘      ← CPU到键盘一根线
  内存───打印机    ← 内存到打印机一根线
  ...
  
  设备多了 → n台设备需要 n(n-1)/2 条线 → 爆炸！💥

总线连接（有总线）：
  所有设备挂到同一条总线上
  
  CPU──┐
  内存──┤
  键盘──┼──→ 公共总线
  鼠标──┤
  网卡──┘
  
  所有设备共用一条总线 → 清爽多了！✅
```

### 8.1.2 总线分类

#### 按连接部件分（最重要的分类）

| 类型 | 连接范围 | 特点 | 例子 |
|:----|:---------|:-----|:------|
| **片内总线** 🔍 | CPU芯片内部寄存器之间 | 速度**最快** | CPU内部数据通路 |
| **系统总线** 🖥️ | CPU ↔ 内存 ↔ I/O接口 | 板级连接 | PCI、ISA |
| **外部总线** 🔌 | 计算机 ↔ 外设 | 速度较慢 | USB、SATA |

#### 按传输方式分

| 类型 | 传输方式 | 优点 | 缺点 |
|:----|:---------|:-----|:------|
| **并行总线** 📏 | 同时传多位数据 | 同频下更快 | 线间干扰、难做高频 |
| **串行总线** 📎 | 一次传1位数据 | 可高频、抗干扰、线少 | 同频下慢 |

> **⚡ 发展趋势：串行代替并行！** 因为串行可以做很高的频率
> 例：PCI（并行33MHz）→ PCIe（串行8GHz）

### 8.1.3 总线的组成

一条总线由三组信号线组成：

| 信号线 | 方向 | 功能 | 类比 |
|:-------|:----:|:------|:------|
| **数据总线** 📊 | **双向** | 传输数据本身 | 高速公路上的"货车"🚛 |
| **地址总线** 📍 | **单向**（CPU→外设） | 传输访问目标地址 | 高速公路的"路牌" |
| **控制总线** 🎛️ | **双向** | 传输读写/中断/仲裁等控制信号 | "红绿灯"🚦 |

**重要规律：**
```
地址总线宽度决定寻址范围
例：32位地址总线 → 可寻址 2³² = 4GB 空间

数据总线宽度决定单次传输量
例：64位数据总线 → 一次可传8字节
```

### 8.1.4 总线性能指标

#### 带宽计算公式 ⭐

```
总线带宽 = 总线宽度 × 时钟频率 × 每时钟传输次数

总线宽度 = 一次传多少位（如32位=4B）
时钟频率 = 总线多快（如33MHz=33×10⁶次/秒）
每时钟传输次数 = SDR=1次, DDR=2次, QDR=4次
```

**计算示例：**
```
例1：PCI总线 32位 33MHz SDR
  带宽 = 4B × 33×10⁶ × 1 = 133 MB/s

例2：PCIe x16 3.0 (8GHz, 128b/130b编码, x16通道)
  每通道带宽 = 8×10⁹ × 128/130 = 7.877 Gb/s ≈ 1 GB/s
  16通道 = 16 GB/s（实际约15.8 GB/s）
```

#### 常见总线带宽对比

| 总线 | 年代 | 类型 | 总线宽度 | 频率 | 带宽 |
|:----|:----:|:----:|:--------:|:----:|:----:|
| **ISA-16** 🐢 | 1981 | 并行 | 16位 | 8.3 MHz | **16 MB/s** |
| **PCI** 📟 | 1992 | 并行 | 32位 | 33 MHz | **133 MB/s** |
| **AGP 8x** 🎮 | 2002 | 并行 | 32位 | 66 MHz QDR | **2.1 GB/s** |
| **PCIe 3.0 x16** 🚀 | 2010 | 串行 | 1位×16通道 | 8 GHz | **~15.8 GB/s** |
| **DDR4-3200** 💾 | 2014 | — | 64位 | 1.6 GHz DDR | **25.6 GB/s** |

## 8.2 🔄 总线传输机制

### 8.2.1 总线传输的四个阶段

```
① 申请阶段 🙋
   设备想用总线 → 向仲裁器发请求
   
② 地址阶段 📍
   获得使用权 → 发出目标地址
   
③ 数据阶段 📦
   传输数据（读或写）
   
④ 结束阶段 🏁
   释放总线 → 下一个设备可以用
```

### 8.2.2 突发传输 (Burst Mode) ⭐

> **一次地址 + 多次数据 = 省去重复发地址的时间**

```
普通传输：
  地址→数据→地址→数据→地址→数据→地址→数据
  A0  D0  A1  D1  A2  D2  A3  D3
  ←每次都要花时间发地址→

突发传输：
  地址→数据→数据→数据→数据
  A0  D0  D1  D2  D3
  ←只发一次地址，后面的数据自动连续排列→
  
块传输效率大幅提升！特别是连续内存访问时
```

### 8.2.3 串行 vs 并行 ⭐

> 为什么现代总线都转向串行？

| 对比项 | 并行 📏 (PCI) | 串行 📎 (PCIe) |
|:-------|:------------|:--------------|
| **传输线数量** | 多（几十根） | **少**（几根差分对） |
| **同频率速度** | 🚀快（同时传多位） | 🐢慢（一次传1位） |
| **可达到的最高频率** | **低**🙅（线间串扰） | **极高**（8GHz+） |
| **抗干扰能力** | 差（相邻线互相干扰） | **强**（差分信号） |
| **传输距离** | 短（几厘米就衰减） | **长**（几米没问题） |
| **成本** | 高（线多、板面积大） | **低**（线少） |
| **发展趋势** | ❌ 逐渐淘汰 | ✅ **主流方向** |

> **⚡ 通俗理解：**
> - 并行 = 10个人并排跑步 → 人多了互相撞 → 跑不快
> - 串行 = 1个人跑 → 没人撞 → 可以跑极快
> - 所以：串行虽然一次只传1位，但频率高→实际更快！

## 8.3 🏁 总线仲裁 ⭐⭐

### 为什么需要仲裁？

```
多个设备想同时使用总线 → 谁先用？
→ 需要仲裁器（裁判）来做决定
```

### 8.3.1 链式查询

```
              ┌────┐    ┌────┐    ┌────┐    
  总线请求 ├──│I/O1│────│I/O2│────│I/O3│──→ …
              │    │    │    │    │    │
  总线允许 ←──┤    ├────┤    ├────┤    ├──→ …
              └────┘    └────┘    └────┘
              BG(总线允许)像链条一样串联

原理：BG信号逐个设备传递，传到谁谁就可以用总线

优点：
  ✅ 控制线少（3根：BS、BR、BG）
  ✅ 设备扩充简单

缺点：
  ❌ 优先级固定（离仲裁器越近优先级越高）
  ❌ 对故障敏感（一个设备坏了，后面的都用不了）
  ❌ 速度慢（信号要串行传递）
```

### 8.3.2 计数器定时查询

```
每个设备有唯一编号
仲裁器用一个计数器从0开始询问："你需要用总线吗？"
设备回答"是"→ 获得使用权

优先级设置：
  每次从0开始 → 固定优先级（编号越小越高）
  从上次停止处开始 → 循环优先级（公平）
  可设置起始值 → 可编程优先级

特点：
  ✅ 优先级可变化（灵活）
  ✅ 不故障敏感
  ❌ 速度慢（要逐个询问）
  ❌ 控制线较多（2+log₂n根）
```

### 8.3.3 独立请求方式

```
每个设备有独立的请求线和允许线
仲裁器根据优先级算法同时处理所有请求

               BR1 ───→│      │
              I/O1 │   │仲裁  │
               BG1 ←───│器    │
                        │(硬   │
               BR2 ───→│件优  │
              I/O2 │   │先级  │
               BG2 ←───│编码  │
                        │器)   │
               BR3 ───→│      │
              I/O3 │   │      │
               BG3 ←───│      │
                        └──────┘

特点：
  ✅ 速度最快（所有请求同时到达仲裁器）
  ✅ 优先级最灵活（硬件可编程）
  ❌ 控制线最多（2n+1根）
  ✅ 最常用！
```

### 8.3.4 三种仲裁方式对比

| 方式 | 控制线数 | 速度 | 优先级 | 故障敏感 |
|:----|:-------:|:----:|:------|:--------|
| **链式查询** ⛓️ | **3根** | 🐢慢 | 固定 | ❌非常敏感 |
| **计数器定时** 🔢 | 2+log₂n | 🐢慢 | 可变化 | ✅不敏感 |
| **独立请求** 🏆 | 2n+1 | 🚀**最快** | **最灵活** | ✅不敏感 |

## 8.4 ⏱️ 总线定时

### 同步总线 🎵

```
所有设备用一个公共时钟

时钟  ┌┴┐┌┴┐┌┴┐┌┴┐┌┴┐┌┴┐
地址  ────┴─────────
数据  ────────┴─────
       ↑在时钟的某个边沿采样

优点：简单、高速
缺点：所有设备必须适配同一时钟频率
      慢设备跟不上 → 需要等待周期
适用：CPU与内存之间（速度相近的设备）
```

### 异步总线 ✋

```
没有公共时钟，用"握手"（请求-应答）

主设备 ──请求──→ 从设备
       ←──应答──
       ──请求──→
       ←──应答──
       每次通信都要握手

优点：不受时钟限制，兼容不同速度的设备
缺点：慢（每次都要等应答）
适用：慢速设备（键盘、鼠标）
```

### 同步 vs 异步

| 对比 | 同步 🎵 | 异步 ✋ |
|:-----|:-------|:--------|
| 时钟 | 有公共时钟 | 无时钟 |
| 控制方式 | 按时钟节拍 | 握手(Req/Ack) |
| 速度 | 🚀快 | 🐢慢 |
| 设备适配 | 需适配总线频率 | ✅可接不同速度设备 |
| 适用 | 高速设备（内存） | 低速设备（I/O） |

## 8.5 🏗️ 总线结构的发展

### 单总线结构

```
                所有设备挂同一条总线
 CPU────内存────I/O1────I/O2────...
                ←──系统总线──→
                
优点：简单、灵活、易扩充
缺点：所有设备争用一条总线 → 慢！
```

### 双总线结构（引入桥接器）

```
CPU───局部总线(高速)───桥接器───I/O总线(慢速)
                          │
                       主存(内存)

高速设备走局部总线（CPU↔内存）
慢速设备走I/O总线
→ 快慢分离，各不干扰 ✅
```

### 三总线结构（南北桥时代）

```
         │←──HOST总线──→│
       CPU             北桥(MCH)────内存
                          │
                     PCI总线 ──── 南桥(ICH)───ISA总线
                          │              │
                    高速设备(显卡)     慢速设备(鼠标)
                         
北桥(MCH)：连接CPU、内存、显卡 = 高速
南桥(ICH)：连接I/O设备、硬盘、USB = 慢速
```

### 现代架构（北桥集成到CPU）

```
          CPU
     ┌────┴────┐
     │         │
  内存       DMI总线
               │
             PCH(平台控制器中心)
               │
         SATA/USB/音频/网卡...
             
北桥功能 → 集成到CPU内部（提高速度）
南桥功能 → PCH芯片取代
CPU和PCH之间用DMI高速总线连接
```

## 8.6 📡 常用总线详解

### PCI总线

```
PCI = Peripheral Component Interconnect
标准：32位/64位，33MHz/66MHz
带宽：133 MB/s ~ 533 MB/s

特点：
✅ 支持总线主控（设备可直接控制总线）
✅ 支持突发传输
✅ 自动配置（即插即用）
❌ 多设备共享带宽 → 瓶颈
❌ 并行 → 频率上不去
```

### PCI Express (PCIe) ⭐

> **PCI的替代者**，但不是"更快的PCI"，而是**全新的串行总线**

| 特性 | 说明 |
|:----|:------|
| **连接方式** | **点对点**串行（不是共享总线！） |
| **通道** | x1/x4/x8/x16 — 多通道可以绑在一起 |
| **编码** | 1.0/2.0: 8b/10b, 3.0+: 128b/130b |
| **每通道带宽** | PCIe 3.0: 约1 GB/s |
| **x16总带宽** | ~15.8 GB/s |

**PCIe vs PCI 关键区别：**
```
PCI是"共享总线"：
  所有设备挂同一条总线 → 一人用大家都等着
  就像一条单车道，大家排队

PCIe是"交换网络"：
  每个设备有自己的专用通道
  就像每个设备都有专用车道 → 同时走！
  通过交换器(Switch)连接
```

### USB (Universal Serial Bus)

| 版本 | 带宽 | 编码 | 特点 |
|:----:|:----:|:----:|:------|
| **USB 1.0** | 12 Mbps | — | 低速，淘汰 |
| **USB 2.0** | 480 Mbps | — | 高速，广泛使用 |
| **USB 3.0** | **5 Gbps** | 8b/10b | 超速，蓝色接口 |
| **USB 3.1** | 10 Gbps | 128b/130b | 超速+ |
| **USB 3.2** | 20 Gbps | — | 双通道 |
| **USB4** | 40 Gbps | — | 兼容Thunderbolt |

**USB特点：**
```
✅ 热插拔（不用关机就能插拔）
✅ 即插即用（自动识别设备）
✅ 最多连接127个设备（通过集线器）
✅ 可供电（5V，最高900mA）
```

## 🧠 本章口诀

```
总线就是高速路，数据地址控制分三路
并行传输线多但慢，串行一根频率高
发展趋势要记住：串行取代并行路！

总线传输四阶段：申请→地址→数据→释放
突发传输效率高，一套地址多数据

仲裁方式有三种：
  链式查询三根线，固定优先级故障敏感
  计数器查灵活变，独立请求最快最稳

同步总线用时钟，高频高效要求高
异步总线握手谈，设备无论快慢都能接

PCI并行共享线，PCIe串行点对点
单通道一车道，x16就是16车道一起跑
点对点专用通道，再也不用互相等！

带宽公式记心中：宽度×频率×倍率
计算之前先换单位，B=字节b=位不要混
```

## ⭐ 必背公式与概念

| 公式/概念 | 说明 |
|:---------|:------|
| `带宽 = 宽度 × 频率 × 每时钟传输次数` | 总线带宽核心公式 |
| 8b/10b编码 | 实际有效带宽 = 理论 × 0.8 |
| 突发传输 | 一次地址+多次数据 |
| 集中式仲裁三方式 | 链式/计数/独立请求 |
| 同步 vs 异步 | 有无公共时钟 |
| PCIe点对点 vs PCI共享 | 核心区别 |
